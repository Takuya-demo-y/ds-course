# Ch.2　データを「見る」技術

### ── 可視化と探索的データ分析（EDA） ──

**本章のゴール**
- グラフの種類と「何を知りたいときに使うか」を理解し、データから仮説を立てられるようになる

```
【Ch.2 でやること】

Ch.1 の集計結果（数値）
        ↓
グラフで「見える化」する（EDA）  ← 今日ここ
        ↓
「この変数が品種の決め手になりそう」という仮説を立てる
        ↓
Ch.3 の機械学習で仮説を検証する
```

---

## なぜグラフが重要なのか

```
【数値だけ見た場合】             【グラフで見た場合（箱ひげ図）】

品種別 alcohol 平均：            wine_0 ：  ══════▓▓▓▓▓══  ← 明らかに高い！
 wine_0 → 13.7                  wine_1 ：  ══▓▓▓▓▓═══════
 wine_1 → 12.3                  wine_2 ：  ════▓▓▓▓═══
 wine_2 → 13.2

「wine_0 がやや高い」            「wine_0 は他の品種と明確に分離している！」
  ← 気づきにくい                   ← 一目でわかる
```

> 📌 **グラフにすることで、数値だけでは見えなかった「差」「分布」「関係性」が一目でわかります。**  
> データを「見える化」することで、説得力のある仮説が立てられます。

---

## EDA（探索的データ分析）とは

```
EDA = Exploratory Data Analysis（探索的データ分析）

【目的】グラフを「作ること」ではなく「仮説を立てること」

【EDA の流れ】

グラフを描く
    ↓
「気づき」を言葉にする
    ↓  例：「wine_0 のアルコールが他の品種より高い傾向がある」
「仮説」に変換する
    ↓  例：「alcohol は品種を分類するのに役立つかもしれない」
Ch.3 の機械学習で検証する
       ↓  確認：特徴量重要度で alcohol がランクインしているか？
```

> 📌 **プロのデータサイエンティストは EDA に全体作業の 3〜4 割をかけます。**  
> モデルより「データ理解」の方が大切だからです。

---

## グラフの種類と使い分け

```
「何を知りたいか？」で選ぶ

1つの変数の分布を見たい       →  ヒストグラム
                                  「値がどこに集中しているか・外れ値の有無」

グループ間を比較したい          →  箱ひげ図
                                  「品種ごとにアルコールの中央値が違うか」

2変数の関係を見たい             →  散布図
                                  「アルコールが高いほどプロリンも高いか」

全変数の関係を一度に見たい      →  相関ヒートマップ
                                  「13列の中でどのペアが最も相関が強いか」
```

> 📌 **「グラフを作ってから考える」ではなく、「何を知りたいかを先に決めてからグラフを選ぶ」が正しい順序です。**

---

## ヒストグラム ── 分布の「形」を読む

```
【パターン①：山型（正規分布）】   【パターン②：右に裾が長い】
     ▌                                ▌
    ▌▌▌                              ▌▌
   ▌▌▌▌▌                            ▌▌▌▌
  ▌▌▌▌▌▌▌                           ▌▌▌▌▌▌▌▌▌
値が中心に集中                     高い値に外れ値が存在する可能性
「安定したデータ」                 「一部が突出している」

【パターン③：2つの山がある】
    ▌       ▌
   ▌▌▌   ▌▌▌
  ▌▌▌▌▌ ▌▌▌▌▌
「2つのグループが混在している」 ← 品種が混じっているかも？
```

> 📌 **ヒストグラムの「形」を見ることで、データの特徴が一目でわかります。**

---

## 箱ひげ図 ── グループ間の「差」を比較する

```
【箱ひげ図の読み方】

      ┌────────────┐
──────┤    箱       ├──────────  ○  ← 外れ値（極端に外れた点）
      └────────────┘
↑          ↑      ↑          ↑
最小値      Q1   中央値       Q3    最大値
          (25%) (50%)       (75%)

箱の中 = データの「真ん中 50%」が収まっている範囲
箱が広い ＝ ばらつきが大きい（値が安定していない）
箱が狭い ＝ ばらつきが小さい（値が安定している）

【品種別に並べると差が見える】

alcohol（品種別）
wine_0 ：      ════▓▓▓▓▓══   ← 全体的に高い・ばらつき小
wine_1 ：  ════▓▓▓▓▓══════   ← 低め・ばらつき大
wine_2 ：    ═══▓▓▓▓▓═════   ← 中程度
```

---

## 散布図 ── 2変数の「関係」を見る

```
【散布図のパターン】

 正の相関                負の相関                相関なし
（右上がりに並ぶ）       （右下がりに並ぶ）       （バラバラ）
    ↑  ●                    ↑ ●                     ↑ ● ●
    │    ●                  │●                      │   ●
    │      ●               │  ●                    │●    ●
    │        ●             │    ●                  │  ●
    └──────────→           └──────────→             └──────────→

片方が増えると            片方が増えると           2変数に関係なし
もう片方も増える          もう片方は減る

【品種ごとに色を変えると…】
各品種が「かたまり」になって見えれば
→「この2変数でグループが分離している」= 分類に使える！という仮説になる
```

---

## 相関ヒートマップ ── 全変数を「一度に」俯瞰する

```
【相関係数とは？】
2つの変数がどれだけ一緒に動くかを −1.0〜+1.0 で表した数値

+1.0 ──────────── 0 ──────────── -1.0
強い正の相関      無関係       強い負の相関
（共に増える）              （一方が増えると他方が減る）

目安：|0.7| 以上 → 強い相関あり
      |0.3| 未満 → ほぼ無関係

【ヒートマップの見方】
              alcohol  proline  flavanoids
alcohol    [  1.00     0.64      0.24  ]
proline    [  0.64     1.00      0.49  ]
flavanoids [  0.24     0.49      1.00  ]

色が濃い（赤に近い） ＝ 相関が強い → 機械学習で効きやすい変数ペア
色が薄い（白・青）   ＝ 相関が弱い → 独立した変数
```

> 📌 **相関が強い変数は「機械学習の特徴量として有効な候補」です（Ch.3 への伏線）。**

---

## Ch.1 との接続 ── 数値 → グラフで仮説を立てる

```
【Ch.1 でわかったこと → Ch.2 で視覚的に確認する】

Ch.1：describe() で proline の std が 314（最大級）
              ↓
Ch.2：ヒストグラムを見ると… 品種によって分布が大きく違う？
              ↓
Ch.2：箱ひげ図を見ると… wine_0 の proline が圧倒的に高い！
              ↓
仮説：「proline は品種を見分けるキー変数かもしれない」
              ↓
Ch.3：特徴量重要度で proline がランクインするか確認する
```

> 📌 **「std が大きい列 → グラフで確認 → 仮説を立てる」この流れが身につくと、分析の精度が格段に上がります。**

## 💬 発問

> 以下の問いについて、隣の人と 2分間話し合ってみてください。

---

**Q1．Ch.1 で集計した「品種ごとのアルコール平均」は数値でした。**  
**それをグラフにすると、どんな「気づき」が加わると思いますか？**

> 💡 **考え方のヒント：**  
> 数値では「wine_0 が 13.7、wine_1 が 12.3」とわかっても「どのくらい差があるか」は伝わりにくいです。  
> 箱ひげ図にすると「ばらつきの大きさ」「外れ値の有無」「品種間の重なり具合」など、数値だけでは見えない情報が加わります。

---

**Q2．「品種 A・B・C を最も分けやすい特徴量はどれか」を調べるには、**  
**どのグラフを使えばよいと思いますか？その理由も教えてください。**

> 💡 **考え方のヒント：**  
> 「グループ間の差を見たい」→ **箱ひげ図**（品種ごとに描いて箱の位置が離れているか確認）  
> 「2変数の関係を見たい」→ **散布図**（品種ごとに色を変えてかたまりが分離するか確認）  
> 「全変数を一度に比較したい」→ **相関ヒートマップ**（どの列が強く関係しているか俯瞰）  
> まず「目的」を決めてからグラフを選ぶのがポイントです。
