### スライド 01-03｜なぜpandasを使うのか

**データ分析に「表計算ソフト」が足りない理由**

| やりたいこと | Excelの限界 | pandasなら |
|-----------|-----------|-----------|
| 大量データの処理 | 数十万行で重くなる・フリーズ | 数百万件でも数秒 |
| 毎月同じ集計を自動化 | 手作業を毎回繰り返す | コードを1回書けばずっと使える |

```
  Barbera    12.4     520      4.95
    ↑行         ↑列
```

**核心メッセージ**
> pandasを使いこなすことは、「データを表として自由に操れる」ということ。

### スライド 01-04｜【概念①】データを手にしたら最初に「形」を確認する

**データを見るとき、最初の問いは「これは何件・何列あるデータか？」**

**3つの問いかけと確認方法**

```
問い1:「このデータは何行・何列ある？」
         → 大きすぎるデータは重い。小さすぎるデータはモデルが学べない

問い2:「各列のデータの型は正しいか？欠損はあるか？」
         → 数値列が文字列になっていたりしないか
         → NaN（空白）がどこかに紛れ込んでいないか

問い3:「値の範囲はおかしくないか？」
         → 年齢がマイナスになっている
         → 売上が異常に大きい/小さい
```

**なぜ最初にこれをやるか**
> いきなり分析やモデルを作ると、後で「そもそもデータが変だった」という事態になる。  
> 実務での時間の無駄を防ぐための習慣。

**補足**
> データサイエンティストは新しいデータをもらったら、まず「眺める」。この習慣が正確な分析の基礎になる。

**スライドノート（講師向け）**
> 「コードの前に、"何を確認するか"を理解させる。コードは次のデモで出てくる。今は3つの問いを頭に入れることが目的」と意識して話す。

### スライド 01-05｜【概念②】欠損値は「見えない地雷」

**現実のデータには必ず「抜け」がある**

**欠損値（NaN）が生まれる理由**

```
① アンケートの未回答        → 「年収」の欄を空白で提出
② センサーの故障・通信エラー  → 計測値が取れなかった期間
③ データ入力ミス           → 転記の際に飛ばしてしまった
④ システム移行時のデータ消失 → 旧データが変換できなかった
```

**欠損値をそのまま放置すると…**

```
計算ができない → NaN + 数値 = NaN（計算が伝染する）
モデルがエラー → scikit-learnは欠損値を受け付けない
平均が狂う   → 件数に含まれないのに全体の傾向がずれる
```

**欠損値への対処方針（どちらを選ぶか）**

| 方針 | 意味 | 使う場面 |
|------|------|---------|
| **削除する** | 欠損のある行ごと捨てる | 欠損が少量・ランダムな場合 |
| **補完する** | 平均値などで埋める | 欠損が多め・削除するとデータが減りすぎる場合 |

**重要な考え方**
> どちらが正解かはデータによって違う。「どちらが適切か」を判断するのがデータサイエンティストの仕事。

**スライドノート（講師向け）**
> 「今日使うワインデータには欠損がない（確認済み）。でも実務では必ず欠損がある。発問：『アンケートデータで年収欄の30%が未回答のとき、削除と補完どちらにしますか？』と問いかけてもよい」

### スライド 01-06｜【概念③】集計で「グループの違い」を数値で掴む

**「品種ごとに平均を出す」とはどういうことか**

**手順のイメージ**

```
ステップ1：「品種」でデータを仕分ける（グループ化）

  Barolo グループ     Grignolino グループ    Barbera グループ
  ─────────────      ─────────────────      ───────────────
  alcohol: 14.2      alcohol: 12.7          alcohol: 12.4
  alcohol: 13.9      alcohol: 12.5          alcohol: 12.1
  alcohol: 14.5      alcohol: 12.9          alcohol: 12.3
       ↓                  ↓                      ↓

ステップ2：各グループの「平均」を計算する

  Barolo: 14.2       Grignolino: 12.7       Barbera: 12.3
```

**なぜ集計が重要か**
> 個々のデータを見ていても傾向はわからない。グループごとに集計することで「Baroloはアルコール度数が高い傾向がある」という発見が生まれる。

**実務での使われ方**

```
「都道府県ごと」の売上平均 → 地域差を発見
「月ごと」の購入率       → 季節性を発見
「年代ごと」の満足度     → ターゲット層を発見
```

**スライドノート（講師向け）**
> 「groupbyの仕組みは"仕分けてから計算する"という2ステップ。コードは次のデモで見る。今は"仕分け→計算"というイメージを頭に入れてもらえればOK」と話す。

### スライド 01-07｜【概念④】フィルタリングで「条件に合うものだけ」取り出す

**「アルコール度数が13.5以上のワインだけ見たい」を実現する**

**考え方：条件でデータを篩（ふるい）にかける**

```
全データ（178件）
    ↓ 「alcohol >= 13.5」という条件でフィルタ
条件を満たす行だけ残る（例：48件）
```

**日常のアナロジー**

```
Excelの「フィルター機能」と同じ
  → 列のプルダウンで「13.5以上」を選ぶのと同じことをコードでやっている
  → コードでやる利点：条件を変えて何度でも自動で実行できる
```

**AND条件（複数の条件）のイメージ**

```
「アルコール度数が高い AND 色が濃い」ワインを探す

    [alcohol >= 13.5]          [color_intensity >= 5.0]
         ↓                              ↓
    48件を通過        AND         56件を通過
                    ↓
            両方を満たす: 31件
```

**スライドノート（講師向け）**
> 「フィルタリングはExcelのフィルター機能と同じ考え方。コードで書くことのメリットは"条件を変えて何度でも実行できる自動化"にある。AND条件は複数条件を組み合わせるだけ」と話す。

### スライド 01-08｜コードで確認する（デモ）

**概念を確認する ― 4ステップをまとめて動かす**

> ここから手を動かします。「さっきの考え方がコードになった」と思いながら実行してください。  
> コードの構文の説明はしません。演習ノートで丁寧に解説しています。

**STEP 1：データの「形」を確認する（概念①）**

```python
import pandas as pd
from sklearn.datasets import load_wine

wine = load_wine()
df   = pd.DataFrame(wine.data, columns=wine.feature_names)
df["品種"] = wine.target

df.head()      # 先頭5行：「どんなデータか」を目で確認
df.info()      # 行数・列数・型・欠損数
df.describe()  # 最小・最大・平均・標準偏差
```

**STEP 2：欠損値を確認する（概念②）**

```python
df.isnull().sum()  # 各列の欠損件数。0なら問題なし
```

**STEP 3：グループ別に集計する（概念③）**

```python
# 品種ごとの平均アルコール度数・プロリン
df.groupby("品種")[["alcohol", "proline"]].mean().round(2)
```

**STEP 4：条件でフィルタリングする（概念④）**

```python
# アルコール度数が13.5以上のワインだけ取り出す
high_alc = df[df["alcohol"] >= 13.5]
print(f"条件を満たすワイン: {len(high_alc)}件")
```

### スライド 01-10｜Ch.1 まとめ

**今日理解した「考え方」と「コード」の対応**

**概念の流れ（これが核心）**

```
① 形を確認する   → 「何件・何列・型・欠損」の4点を確認
       ↓
② 欠損値を処理  → 削除か補完か、データに合わせて判断
       ↓
③ 絞り込む     → 条件に合う行だけを取り出す
       ↓
④ 集計する     → 仕分けてから計算。グループの差を数値で掴む
```

**コマンドリファレンス**

| 操作 | コード | ポイント |
|------|--------|---------|
| 形を確認 | `head() / info() / describe()` | この3つをセットで実行 |
| 欠損確認 | `isnull().sum()` | 0なら問題なし |
| 絞り込み | `df[df["列"] >= 値]` | Excelのフィルターと同じ |
| 集計 | `groupby("列").mean()` | 仕分けてから計算 |

**核心メッセージ**
> 🔑 コードより大切なのは「何を確認するか」という問いの立て方。  
> 形・欠損・絞り込み・集計の4ステップは、どんなデータでも共通の入口。

**Ch.2への橋渡し**
> ➡️ 次のCh.2では、集計した数値を「グラフ」にして、仮説を立てます。  
> 「Baroloのアルコール度数が高い」という数値を、目で見てわかる形にします。
