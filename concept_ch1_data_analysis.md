# Ch.1　データを「読む」技術

### ── pandas によるデータ読み込みと集計 ──

**本章のゴール**
- 集計・フィルタリングで「知りたいことを数値で答える」感覚を掴む

```
【Ch.1 でやること】

データを受け取る
     ↓
STEP 1：形を把握する  → 何行・何列？欠損は？型は？
     ↓
STEP 2：集計する      → グループ別に平均・合計を出す
     ↓
STEP 3：絞り込む      → 条件に合う行だけ取り出す
     ↓
「知りたいことを数値で答える」
```

---

## なぜデータ分析が必要か ── 実務での使い場面

```
【今まで（勘）】                【データ分析後（根拠あり）】
「なんとなく A 商品が            「A 商品は関西で先月比 -30%。
  売れていると思う」               他地域は +5%」
       ↓                               ↓
  会議で「なんとなく」話す          数字を根拠に具体策を議論できる
```

| シーン | やること | 出てくるアウトプット |
|-------|---------|-------------------|
| 営業会議の前 | 商品別・地域別の売上を集計する | 「関西の A 商品が先月比 -30%」 |
| 品質レポート | ラインごとの不良品発生率を確認する | 「ライン3の不良率が突出して高い」 |
| 顧客分析 | 年齢・性別・購買金額でセグメントを確認 | 「40代女性の客単価が最大」 |

> 📌 **「数字で質問に答える」のがデータ分析の本質。今日使うワインデータも考え方は同じです。**

## データの形（DataFrame）を理解する

```
【DataFrame ＝ 表形式のデータ】

          alcohol  malic_acid  ...  proline  品種名
行 0  →  14.23      1.71     ...   1065    wine_0
行 1  →  13.20      1.78     ...   1050    wine_0
行 2  →  13.16      2.36     ...   1185    wine_0
  ︙         ︙          ︙     ...      ︙       ︙
行 177 →  14.13      4.10     ...    495    wine_2
    ↑
  178行（件数）× 14列（特徴量 13列 + 品種名 1列）

列 = 特徴量（変数）：各ワインの化学成分の測定値
行 = 1件分のデータ：1本のワインの測定結果
```

> 📌 **Excel の「シート」と同じ構造です。行 = 1件のデータ、列 = 項目（変数）。**

---

## STEP 1　形を把握する ── 最初に必ず確認する5つ

| 確認すること | 使う命令 | 何がわかるか |
|------------|---------|-----------|
| **行数・列数** | `df.shape` | 分析の規模（178行×14列） |
| **先頭5行の中身** | `df.head()` | どんな値が入っているか目で確認 |
| **列名・データの型** | `df.info()` | 数値型か文字列型かで使える操作が変わる |
| **欠損値（空欄）の数** | `df.isnull().sum()` | 欠損があると集計が狂う |
| **統計量** | `df.describe()` | 平均・最大・最小・std（ばらつき） |

> 📌 **この5つを飛ばして分析を始めると、後で「欠損に気づいていなかった」「型が違った」という事故が起きます。**

---

## STEP 1　`describe()` の読み方

```
           alcohol   proline
count    178.000    178.000   ← データの件数（欠損があるとここが減る）
mean      13.000    746.893   ← 平均値
std        0.812    314.907   ← 標準偏差（ばらつきの大きさ）
min       11.030    278.000   ← 最小値
25%       12.362    500.500   ← 下位25%の値
50%       13.050    673.500   ← 中央値（真ん中の値）
75%       13.678    985.000   ← 上位25%の値
max       14.830   1680.000   ← 最大値
```

### std（標準偏差）に注目する理由

```
alcohol の std ＝  0.8   → 値がほとんど 12〜14 の間に収まっている（ばらつき小）
proline の std ＝ 314.9  → 値が 278〜1680 と大きく散らばっている（ばらつき大）
                    ↓
       std が大きい列 ＝ グループ間で差が出やすい ＝ 分析で着目すべき変数の候補
```

> 📌 **Ch.3 で機械学習をしたとき、std が大きい `proline` が「効いた変数」になるかを確認しましょう。**

## 欠損値（NaN）とは ── データの「空欄」

```
【欠損値がある場合の問題】

氏名      年齢    売上
田中       32     120万  ← 正常
鈴木      NaN     98万   ← 年齢が空欄（欠損）
佐藤       45     NaN    ← 売上が空欄（欠損）

↓ 欠損に気づかず平均を計算すると…

年齢の平均：(32 + 45) ÷ 2 = 38.5  （本来は3人で割るべきだが2人分しかない）
                 ↓
           正しくない平均が出てしまう！
```

### 対処法（2択）

- **削除**：欠損のある行を除外する（件数が少ない場合）
- **補完**：平均値・中央値などで埋める（件数を減らしたくない場合）

> 📌 **今日のワインデータに欠損はありません。でも実務データには必ずあります。**  
> 欠損値を残したまま機械学習モデルに渡すと **fit() がエラーになります**（Ch.3 で確認）。Ch.1 の `isnull().sum()` 確認が、その防止の第一歩です。

## STEP 2　グルーピング集計

```
【全データを「品種」でグループに分けて計算する】

全データ（178件）
     │
     ├─ wine_0（59件） → アルコールの平均を計算 → 13.7度
     ├─ wine_1（71件） → アルコールの平均を計算 → 12.3度
     └─ wine_2（48件） → アルコールの平均を計算 → 13.2度

Python コード：df.groupby("品種名")["alcohol"].mean()
Excel の操作：ピボットテーブル で「品種名」を行ラベル・「平均」で集計
                     ─────── 考え方はまったく同じ ───────
```

| 実務の基準 | 計算 | 知りたいこと |
|-----------|------|------------|
| 部署別 | 売上の合計 | どの部署が最も貢献しているか |
| 製造ライン別 | 不良品数の平均 | どのラインの品質が悪いか |
| 品種別 | アルコール・プロリンの平均 | **今日の演習はここ** |

---

## STEP 3　フィルタリング ── 条件で行を絞り込む

```
【全データから「条件に合う行だけ」取り出す】

全データ（178件）
     │
     │  条件：アルコール ≥ 13.5 かつ プロリン ≥ 700
     │
     ↓
条件を満たす行だけ取り出す（例：42件）
     ↓
この 42件だけで詳細分析する

Python コード：df[(df["alcohol"] >= 13.5) & (df["proline"] >= 700)]
Excel の操作：フィルター機能で「13.5以上」を選択
                     ─────── 考え方はまったく同じ ───────
```

| 実務の使い方 | フィルタリングの条件 |
|-----------|-----------------|
| 問題ラインの特定 | 不良品数 ≥ 5件 のラインのみ抽出 |
| 顧客セグメント分析 | 年齢 40〜50歳 かつ 購買金額 ≥ 10万円 |
| 異常データの抽出 | センサー値が正常範囲を外れた行のみ |

## 演習との接続 と まとめ

### 今日の演習でやること

| ステップ | 内容 | 注目ポイント |
|---------|------|-----------|
| **STEP 1** | ワインデータの形を確認する | `describe()` で std が大きい列を探す |
| **STEP 2** | 品種ごとのアルコール・プロリン平均を集計 | 品種間で差が大きい列 ＝ Ch.3 で「効く」変数の候補 |
| **STEP 3** | 条件で絞り込み、品種の内訳を確認 | どの品種に偏るか |

### Ch.1 で学んだこと

| 概念 | 内容 |
|------|------|
| **DataFrame** | 行 = 1件のデータ、列 = 特徴量（Excel の表と同じ） |
| **STEP 1：形を把握** | shape / head / info / isnull / describe の5点確認 |
| **STEP 2：集計** | groupby で「グループ別の平均・合計」を計算（ピボットと同じ） |
| **STEP 3：フィルタリング** | 条件に合う行だけ抽出（Excel フィルターと同じ） |

### 次の Ch.2 へ

> 集計で出た数値は表でも見られますが、**グラフにすると何倍も伝わりやすく、仮説が立てやすくなります。**
