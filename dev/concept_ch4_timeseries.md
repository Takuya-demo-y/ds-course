# Ch.4　異常を「検知」する技術

## 今日の問い：機械はどうやって「普通じゃない値」に気づくのか

---

## 異常検知とは何か ── 「正常なパターン」との差を測る

```
【異常検知の考え方】

  Step 1：正常なデータだけを集める（ベースライン）
              ↓
  Step 2：そのデータからモデルを学習させ「正常の型」を覚えさせる
              ↓
  Step 3：新しいデータに対して「型に当てはめる」
              ↓
  Step 4：型から大きく外れた点 ＝ 異常候補（異常スコアが高い）
              ↓
  Step 5：しきい値を決めて「異常フラグ」を立てる
```

**異常検知が使われている実務場面**

| 場面 | 検知したい異常 |
|------|------------|
| **製造設備の保守** | センサーの振動・温度が正常範囲を外れる（故障の前兆） |
| **インフラ監視** | サーバーの処理量が突然急増する（障害・攻撃の兆候） |
| **品質管理** | 製品の測定値が規格外になる（不良品の混入） |

> 📌 **Ch.3 で学んだ「正解ラベルがある」教師あり学習と同じ枠組みで実現できます。**
> ただし「異常ラベルは使わない」──正常データだけで「正常の型」を作ります。

## 回帰問題とは何か ── 数値を予測する問題

```
【分類と回帰の違いを振り返る】

         分類（Ch.3）                 回帰（Ch.4）
  ──────────────────────           ──────────────────────
  出力がカテゴリ                    出力が数値
  wine_0 / wine_1 / wine_2         センサー値：24.3 / 25.1 / 23.8 …
  良品 / 不良品                     次の時刻の温度：78.2 度

  問い：「どのクラスか？」           問い：「どのくらいの値か？」
```

```
【回帰でできること（時系列への応用）】

  過去の値（特徴量 X）  →  次の時刻の値（y）を予測
        ↓
  予測値と実際値の「誤差（残差）」を計算
        ↓
  残差が大きい ＝ 予測から大きく外れた ＝ 異常の可能性
```

> 📌 **「予測して、外れた点を異常とみなす」これが今日の異常検知の中心的なアイデアです。**
> Ch.3 の `fit → predict` の流れをそのまま使います。

## 📎 補足｜回帰アルゴリズムの系譜 ── ポジションマップ

```
【主な回帰アルゴリズムの位置づけ】

精度
高│                              ★ XGBoost（回帰）
  │                  ★ ランダムフォレスト（回帰）
  │
  │  ★ 線形回帰  ← 今日使うのはここ（シンプルで解釈しやすい）
  │
低└──────────────────────────────────────── 解釈しやすさ
   低（複雑で説明しにくい）            高（シンプルで説明しやすい）
```

| アルゴリズム | 特徴 | 向いている場面 |
|-----------|------|-------------|
| **線形回帰** | y = ax + b の形で予測。係数を見れば各変数の影響がわかる | まず試す baseline・解釈が重要な場面 |
| **ランダムフォレスト（回帰）** | 高精度・非線形関係も捉えられる | 精度優先の実務タスク |
| **XGBoost（回帰）** | 最高水準の精度 | コンペ・精度最優先の場面 |

> 📌 **今日の演習では最もシンプルな「線形回帰」を使います。**
> 「1つ前の値から次の値を予測する」という単純な関係を最初に学習させます。

---

## 回帰の評価指標 ── 誤差をどう測るか

```
【予測と実際の「ズレ」を測る2つの指標】

  実際の値：24.3 度
  予測の値：24.0 度
  残差（誤差）：0.3 度  ← この「ズレ」を評価する

────────────────────────────────────────────────────────

  MAE（平均絶対誤差）
  「全データの誤差の絶対値を平均したもの」
  → 単位は元データと同じ（例：0.5 度）
  → 直感的にわかりやすい

  R²（決定係数）
  「モデルがデータのばらつきをどれだけ説明できているか」
  → 0〜1 の範囲（1に近いほど良い・0はランダムと同程度）
  → "当てはまりの良さ"を示す
```

> 📌 **異常検知では「残差の絶対値（誤差の大きさ）」こそが異常スコアになります。**
> R² で全体の精度を確認しつつ、各点の残差で「どこが異常か」を特定します。

## 時系列データとは何か ── データの「順序」が意味を持つ

```
【通常のデータと時系列データの違い】

  通常のデータ（ワインデータなど）
  ───────────────────────────────
  行の順番に意味はない
  サンプル A を上にしても下にしても分析結果は変わらない

  時系列データ（センサーデータなど）
  ───────────────────────────────
  行の順番 ＝ 時刻の順番
  9:00 → 9:01 → 9:02 … と時間が進む
  順番を変えると「過去・未来」の関係が崩れる
```

```
【時系列データ特有の注意点】

  ⛔ シャッフルしてはいけない（順番に意味があるため）
  ⛔ 訓練とテストは「時刻で」分割する（前半＝訓練、後半＝テスト）
     ※ ランダム分割すると「未来のデータ」で学習することになる（リーク）
  ✅ グラフで時系列の傾向・周期・異常を目視確認することが大切
```

> 📌 **「時刻の順番を守ること」が時系列分析の絶対ルールです。**

---

## 「1つ前の値」を手がかりにする ── 特徴量エンジニアリング

```
【ラグ特徴量（lag feature）の考え方】

  時刻 t の値を予測したい
  → 「時刻 t-1 の値（1つ前）」を手がかりとして使う

  時刻    実際の値   ラグ（1つ前の値）   → 予測対象
  ─────────────────────────────────────────
  9:00      24.1        （なし）
  9:01      24.3         24.1           → 9:01 の値を予測
  9:02      24.0         24.3           → 9:02 の値を予測
  9:03      25.8         24.0           → 9:03 の値を予測
  9:04      25.6         25.8           → 9:04 の値を予測
```

```
【なぜ「1つ前の値」で予測できるのか】

  正常なセンサーデータは「なめらかに変化する」
  → 今の値は「直前の値とほぼ近い」はず
  → 直前の値から大きく外れている ＝ 急激な変化 ＝ 異常の疑い
```

> 📌 **「過去の情報から未来を予測する」これが時系列の特徴量エンジニアリングの核心です。**

---

## なぜ「正常期間だけ」で学習するのか ── ベースラインの考え方

```
【正常期間の学習 → 異常期間のテスト】

  全期間のデータ
       │
       ├── 前半（正常期間）→ 「正常のパターン」を学習させる
       │     ↑
       │   正常なデータだけでモデルを作る ＝ 正常の「型」
       │
       └── 後半（テスト期間）→ 学習した「型」で予測する
             ↑
           「型から外れた点」を異常として検知する
```

```
【もし異常データも学習に含めたら…】

  異常 → 正常と一緒に「学習」してしまう
  → モデルが「異常も含めた平均」を覚えてしまう
  → 異常が起きても「想定内」として見逃してしまう
  → 異常の検知率が大幅に下がる
```

> 📌 **「正常の型をきっちり学習させるために、正常期間だけを使う」**
> これは医療の「健康診断の基準値」を健康な人のデータだけで作るのと同じ考え方です。

## しきい値のトレードオフ ── 見逃し vs 誤検知

```
【異常スコア（残差の大きさ）としきい値の関係】

  残差（誤差）
   大 │          ●                         ● ← 異常スコアが高い点
      │     ●       ●                ●
      │  ●               ●      ●
      │●                       ●           ●
   小 │────────────────────────────────────
      時刻 →

  しきい値を低く設定（敏感）：多くの点を検知 → 誤検知が増える
  しきい値を高く設定（鈍感）：見逃しが減る → 本物の異常も見逃す
```

**パーセンタイルで「統計的に外れた点」を検知する**

| しきい値の決め方 | 意味 |
|--------------|------|
| **95パーセンタイル** | 残差の上位5%を異常とみなす（約20件に1件を検知） |
| **99パーセンタイル** | 残差の上位1%を異常とみなす（約100件に1件を検知） |

> 📌 **「どこで線を引くか」は統計ではなくビジネスの判断です。**
> 見逃しのコスト vs 誤検知のコストを比較して決めます（Ch.3 の Precision/Recall と同じ考え方）。

**あなたの業務のデータを時系列で並べると、**
**「正常なパターン」と「異常な変化」はどう見えるでしょうか？**

> 💡 **考え方のヒント：**
> ・「急激に上がった・下がった」＝ 急変型の異常
> ・「ゆっくりと通常の範囲を超えていった」＝ 漸進型の異常（設備の劣化など）
> ・「いつも1〜5時は低い値なのに今日は高い」＝ 周期パターンからの逸脱
>
> 正常とはどんな状態か、その「型」を言葉で表現できることが
> 異常検知設計の第一歩です。
