# Ch.2　データを「見る」技術

## 今日の問い：数字をグラフにすると、何が見えてくるのか

---

## なぜグラフで見るのか ── 数字だけでは伝わらない3つの情報

```
【数字（表）だけで見た場合】        【グラフで見た場合（箱ひげ図）】

品種別 アルコール平均               wine_0 ：  ════▓▓▓▓▓══  ← 明らかに高い！
  wine_0 → 13.7                    wine_1 ：  ══▓▓▓▓▓════
  wine_1 → 12.3                    wine_2 ：  ═══▓▓▓▓══
  wine_2 → 13.2

「wine_0 がやや高い」              「wine_0 は他の品種と明確に分離している」
  ← 差の大きさが伝わらない            ← ばらつきの違いまで一目でわかる
```

**グラフで初めて見えてくる3つの情報**

| 情報 | 内容 | 数字だけでわかるか |
|------|------|-----------------|
| **分布** | 値がどこに集まっているか・どれだけ散らばっているか | ❌ わかりにくい |
| **差** | グループ間でどのくらい違うか・重なっているか | ❌ わかりにくい |
| **関係** | 2つの変数が連動しているか | ❌ わかりにくい |

> 📌 **「数字を見て → グラフで確かめる」この組み合わせが分析の基本です。**

---

## 分布の「形」を読む ── ヒストグラム

```
【ヒストグラム ＝ 「値がどこに、どのくらい集まっているか」を見る】

パターン①：山型（正規分布）       パターン②：右裾が長い
      ██                                  ██
    ██████                              ██████
  ██████████                          ████████████
値が中心に集中                       高い値に少数の外れ値がある可能性
「安定したデータ」                   「一部が突出している」

パターン③：2つの山がある
    ██          ██
  ██████      ██████
「2つのグループが混在している」 ← 品種が混じっているかも？
```

**ヒストグラムで確認するポイント**

| 確認項目 | 意味 |
|---------|------|
| **山の位置（中心）** | データが集まっている値の範囲 |
| **山の広がり（幅）** | ばらつきの大きさ。広いほど値が安定していない |
| **裾の長さ** | 外れ値や少数の突出した値の存在 |
| **2つの山** | 異なる性質のグループが混在している可能性 |

> 📌 **演習では「品種ごとに色分けして重ねる」と品種間の分布の違いが一目でわかります。**

---

## グループ間の「差」を見る ── 箱ひげ図

```
【箱ひげ図の読み方】

        ┌──────────────┐
────────┤      箱       ├────────────  ○  ← 外れ値
        └──────────────┘
↑              ↑       ↑            ↑
最小値          Q1   中央値（50%）   Q3    最大値
             (25%)                (75%)

  箱の中  ＝ データの「真ん中 50%」が収まっている範囲
  箱が広い ＝ ばらつきが大きい（値が安定していない）
  箱が狭い ＝ ばらつきが小さい（値が安定している）
```

```
【品種別に並べると「差」が見える】

alcohol（品種別）
wine_0 ：      ════▓▓▓▓▓══   ← 全体的に高い・ばらつき小
wine_1 ：  ════▓▓▓▓▓═══════   ← 低め・ばらつき大
wine_2 ：    ═══▓▓▓▓▓═════   ← 中程度

「箱の位置が離れている変数」＝ 品種を区別するのに役立つ変数の候補
```

> 📌 **Ch.3 の機械学習では「どの変数が品種の分類に効くか」を自動で見つけます。**
> 箱ひげ図は「どの変数が有望か」を事前に目で確認するためのツールです。

---

## 2変数の「関係」を見る ── 散布図

```
【散布図のパターン】

正の相関                  負の相関                 無関係
（右上がり）               （右下がり）               （バラバラ）
↑  ●                      ↑ ●                       ↑  ●  ●
│    ●                    │●                         │    ●
│      ●                  │  ●                       │ ●      ●
│        ●                │    ●                     │    ●
└──────────→              └──────────→               └──────────→

片方が増えると             片方が増えると             2変数に関係なし
もう片方も増える           もう片方は減る
```

```
【品種ごとに色を変えると「分離」が見える】

 proline
 高│        ● ● ●  ← wine_0（右上に集まる）
   │      ●●●●●
   │    ■■■■       ← wine_1（左下に集まる）
 低│  ■■■●■
   └─────────────────── alcohol
    低               高

→「この2変数のペアが品種を区別できる」という仮説が立てられる
```

> 📌 **品種ごとの点群が「かたまり」になって分離しているほど、分類に使いやすい変数ペアです。**

---

## 全変数の関係を一度に俯瞰する ── 相関ヒートマップ

```
【相関係数とは ── 2変数がどれだけ一緒に動くかを −1.0〜+1.0 で表した数値】

+1.0 ──────────── 0 ──────────── -1.0
強い正の相関      無関係       強い負の相関
（共に増える）              （一方が増えると他方が減る）

目安：｜値｜≥ 0.7 → 強い相関あり
      ｜値｜＜ 0.3 → ほぼ無関係
```

```
【ヒートマップの見方（色で相関の強さを読む）】

              alcohol  proline  flavanoids  malic_acid
alcohol    [  1.00     0.64      0.24       -0.11  ]
proline    [  0.64     1.00      0.50       -0.19  ]
flavanoids [  0.24     0.50      1.00       -0.41  ]
malic_acid [ -0.11    -0.19     -0.41        1.00  ]

色が濃い赤（＋1.0 に近い）＝ 強い正の相関
色が濃い青（－1.0 に近い）＝ 強い負の相関
色が白に近い（0 に近い）  ＝ ほぼ無関係
```

> 📌 **まず「色が濃い場所」を探すだけで、変数間の重要な関係が見えてきます。**

---

## 相関が強い変数には注意が必要 ── 多重共線性

```
【多重共線性 ＝ 特徴量どうしが強く相関している状態】

例）ある製品の品質を予測するモデル

  特徴量として「温度」と「熱量」の両方を使うとする
          ↓
  「温度が高い ＝ 熱量も高い」（ほぼ同じ情報）
          ↓
  モデルは「温度と熱量、どちらを信じればいいか」混乱する
          ↓
  予測が不安定になる・重要な変数が正しく評価されなくなる
```

**多重共線性が起きると何が問題か**

| 問題 | 内容 |
|------|------|
| **モデルが不安定** | 学習のたびに結果が大きく変わる |
| **重要度の誤評価** | 本来重要な変数が低く評価される |
| **解釈が難しくなる** | 「この変数が効いた」という判断ができない |

> 📌 **相関が強い変数ペアを見つけたら「どちらか一方を使う」という判断が必要になります。**
> ヒートマップはその判断をするための地図です。

## 📎 補足｜スケーリングとは何か ── 変数の「単位の差」を揃える

```
【変数によってスケール（単位の大きさ）が全然違う問題】

  alcohol  ：  11〜15 の範囲（幅は約 4）
  proline  ：  278〜1680 の範囲（幅は約 1400）

  モデルは「数値の差が大きいほど重要」と判断しやすい
          ↓
  proline は alcohol の 350倍もスケールが大きい
          ↓
  何もしないと proline ばかりが結果に影響してしまう
  （alcohol が本当は重要でも、埋もれてしまう）
```

```
【標準化（Standardization）── スケールを揃える代表的な方法】

  各値から平均を引いて、標準偏差で割る
          ↓
  どの変数も「平均 0・標準偏差 1」のスケールに揃う
          ↓
  alcohol も proline も同じ土俵で比較できるようになる
```

> 📌 **今日の演習データ（ワイン）では単位差の影響が少ないため、スケーリングは行いません。**
> 実務では「変数のスケールが大きく違う場合」に必ず検討します。

## EDAの目的は「仮説を立てること」

```
【EDA（Exploratory Data Analysis：探索的データ分析）の流れ】

グラフを描く
      ↓
「気づき」を言葉にする
      ↓  例：「wine_0 のアルコールが他の品種より高い傾向がある」
「仮説」に変換する
      ↓  例：「alcohol は品種を分類するのに役立つかもしれない」
Ch.3 の機械学習で検証する
      ↓  確認：特徴量重要度で alcohol がランクインしているか？
```

**大切なのはグラフを「作ること」ではない**

| ❌ やってしまいがちなこと | ✅ 本来の目的 |
|----------------------|------------|
| とにかくグラフを量産する | 「知りたいこと」を先に決めてからグラフを選ぶ |
| グラフを見て「きれいだな」で終わる | グラフから「なぜこうなっているか」を考える |
| 気づいたことを言葉にしない | 「気づき」を文章で記録し仮説にする |

> 📌 **プロのデータサイエンティストは EDA に全作業の 3〜4 割をかけます。**
> モデルより「データ理解」の方が大切だからです。

---

## Ch.2 → Ch.3 への橋渡し ── 仮説を機械学習で検証する

```
【Ch.2 でやること → Ch.3 で確かめること】

Ch.2 でわかること                    Ch.3 で確認すること
──────────────────────               ────────────────────────────
ヒストグラム                          →
「proline の分布が品種ごとに大きく違う」   「proline は特徴量重要度で上位か？」

箱ひげ図                             →
「alcohol は wine_0 だけ明確に高い」     「alcohol は分類に貢献しているか？」

散布図                               →
「alcohol × proline で品種が分離する」   「この2変数の組み合わせは有効か？」

相関ヒートマップ                      →
「flavanoids と nonflavanoid_phenols   「強く相関する変数は重複して
 が強く相関している（−0.76）」           選ばれていないか？」
```

> 📌 **Ch.3 の演習が終わったら「自分の仮説は合っていたか」を必ず振り返りましょう。**
> 「仮説 → 検証 → 振り返り」がデータ分析の核心的な思考サイクルです。
