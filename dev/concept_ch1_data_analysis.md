## なぜデータで考えるのか

```
【今まで（勘・経験）】             【データ分析を使うと】

「なんとなく A ラインの            「A ラインは今月の不良率が
 品質が落ちている気がする」         B ラインの 3 倍。
        ↓                          先月から上昇傾向にある」
  会議で「たぶん…」と話す                  ↓
                                   数字を根拠に具体策を議論できる
```

| 業務の場面 | 「勘」での判断 | データがあると |
|-----------|-------------|--------------|
| 売上が落ちた | 「景気のせいかも」 | 「関西の A 商品が先月比 −30%」 |
| 品質トラブル | 「ラインが疲弊している」 | 「ライン 3 の不良率が突出して高い」 |
| 顧客戦略 | 「若い人向けにしよう」 | 「40代女性の客単価が最大」 |

> 📌 **「なんとなく」を「数字」に変えることがデータ分析の本質です。**

## データとは何か ── 特徴量（X）とターゲット（t）

```
【データ = 表（行と列）の集まり】

         alcohol  proline  flavanoids  ash  ...   品種名
行 0  →   14.2    1065      2.8       2.4  ...   wine_0  ┐
行 1  →   13.2    1050      2.5       2.1  ...   wine_0  │ 特徴量 X
行 2  →   12.4     500      1.2       1.9  ...   wine_1  │（手がかり）
  ︙        ︙       ︙        ︙        ︙    ...      ︙   ┘
                                                  ↑
                                          ターゲット t（予測したい値）
```

| 用語 | 意味 | 今日の例 |
|------|------|---------|
| **特徴量（X）** | モデルへの「入力」となる手がかり | alcohol・proline など 13列 |
| **ターゲット（t）** | 予測したい「答え」 | 品種名（wine_0 / 1 / 2） |

> 📌 **行 = 1件のデータ、列 = 特徴量。Excel のシートとまったく同じ構造です。**

## データを受け取ったら確認する5つの問い

```
【分析を始める前に必ず確認する5つの問い】

① 何件のデータがあるか？
      → 分析の規模を把握する（少なすぎると結論が出ない）

② 実際にどんな値が入っているか？
      → 先頭の数行を目で見て確認する

③ 各列の型は何か？（数値か・文字か）
      → 型が違うと計算できない

④ 欠損（空欄）はあるか？
      → 空欄に気づかないまま集計すると結果が狂う

⑤ 数値の広がりはどうか？（平均・最大・最小・ばらつき）
      → 極端な値がないか、ばらつきが大きい列はどれか
```

> 📌 **この5点を飛ばすと「欠損に気づいていなかった」「型が違った」という事故が起きます。**
> 演習では実際にこの順番で確認していきます。

## 欠損値とは何か ── 「空欄」が分析を狂わせる

```
【欠損値 = データの「空欄」】

  氏名      年齢    月次売上
  田中       32     120万    ← 正常
  鈴木      ___      98万    ← 年齢が空欄（欠損）
  佐藤       45     ___      ← 売上が空欄（欠損）

↓ 欠損に気づかないまま「年齢の平均」を計算すると…

  本来は3人分の平均を出すべきなのに
  2人分（32歳・45歳）だけで計算してしまう
  → 「平均年齢 = 38.5歳」という誤った数字が出る

  さらに機械学習モデルにこのまま渡すと → エラーが発生する
```

> 📌 **欠損値は「0」ではなく「値がない状態」です。0 と空欄は別物です。**

## 📎 補足｜欠損値の処理方法 ── 補完と削除

```
【欠損値への2つのアプローチ】

方法①：補完（値を埋める）
  空欄を「平均値」や「中央値」で埋める
  ✅ 件数が減らない
  ⚠️ ただし「作った数字」が入るため精度に影響する可能性がある

  例）年齢の欠損 → その列の平均年齢（38.5歳）で補完する

方法②：削除（その行ごと除外する）
  欠損のある行を取り除く
  ✅ 実測値だけが残るので信頼性が高い
  ⚠️ ただしデータ件数が減る

  例）年齢が空欄の鈴木さんの行を丸ごと除外する
```

| 状況 | 推奨する方法 |
|------|------------|
| データ件数が十分多い | 削除（シンプルで安全） |
| データ件数が少ない | 補完（件数を維持する） |

## 外れ値とは何か ── 「飛び抜けた値」が平均を狂わせる

```
【外れ値 = 他のデータから極端に離れた値】

例）ある部署の月収データ（7人）

  25万、28万、26万、27万、24万、26万、2000万

                            ↑ 突出した1件（経営者が含まれていた）

↓ 外れ値を含めたまま「平均月収」を計算すると…

  (25+28+26+27+24+26+2000) ÷ 7 ≒ 308万円

  ← 残り6人は全員 30万円以下なのに「平均 308万」は実態と大きくかけ離れている
```

**外れ値が発生する主な原因**

| 原因 | 例 |
|------|---|
| 入力ミス | 「26」と入力すべきところを「260」と入力 |
| 異なるグループの混入 | 工場ラインのデータに試験稼働時のデータが含まれていた |
| 本物の異常 | 製造設備が実際に誤作動した記録 |

> 📌 **外れ値は「削除すべき悪者」ではなく「調査すべき重要なシグナル」の場合もあります。**

## 📎 補足｜外れ値の除去手法 ── 3σ法・Hampel 判別法

```
【代表的な外れ値の判定方法】

① 3σ法（スリーシグマ法）
   「平均から標準偏差の3倍以上離れた値」を外れ値とみなす

   正常範囲 ＝ 平均 ± 3 × 標準偏差

   例）平均 26万、標準偏差 1.5万 の場合
       正常範囲 ＝ 21.5万 〜 30.5万
       → 2000万は明らかに範囲外 → 外れ値と判定

   前提：データが「山型（正規分布）」に従う場合に有効

② Hampel 判別法
   「中央値からの外れ具合」を基準にする
   → データが山型でない場合や、外れ値が多い場合にも頑健
```

> 📌 **どちらの方法も「どのくらい外れたら外れ値か」という基準を決める考え方です。**
> 実務では文脈に応じてしきい値を調整します。

## グループ別に比べるとは ── ピボットテーブルと同じ発想

```
【全データを「グループ」に分けて、それぞれの特徴を数値で比べる】

全データ（178件）
  │
  ├─ wine_0（59件） → アルコールの平均 → ?度
  ├─ wine_1（71件） → アルコールの平均 → ?度
  └─ wine_2（48件） → アルコールの平均 → ?度

「全体の平均」だけでは品種ごとの違いが見えない
「品種ごとの平均」を出して初めて差が見える
```

| 考え方 | データ分析 | Excel |
|--------|-----------|-------|
| グループ別に集計する | グループ別集計 | ピボットテーブル |

**実務での活用例**

| 集計の視点 | 知りたいこと |
|-----------|------------|
| ライン別の不良率 | どのラインに問題があるか |
| 地域別の売上 | どの地域が伸びているか |
| 年代別の購買金額 | どの年代が最も収益に貢献しているか |

> 📌 **演習では「品種ごとのアルコール・プロリンの平均」を集計して品種の特徴を掴みます。**

## 条件で絞り込むとは ── フィルターと同じ発想

```
【全データの中から「条件に合う行だけ」取り出す】

全データ（178件）
  │
  │  条件：「アルコール度数が高い」かつ「プロリンが多い」
  │
  ↓
条件を満たす行だけ取り出す（例：42件）
  ↓
この42件だけを詳しく分析する
```

| 考え方 | データ分析 | Excel |
|--------|-----------|-------|
| 条件に合う行を抽出する | フィルタリング | フィルター機能 |

**実務での活用例**

| 絞り込みの条件 | 目的 |
|-------------|------|
| 不良品が発生した製造日だけ | トラブルの原因を絞り込む |
| 購買金額が10万円以上の顧客だけ | 高付加価値顧客の行動を分析する |
| センサー値が正常範囲を外れた時刻だけ | 異常が起きた瞬間を特定する |

> 📌 **演習では「アルコールとプロリンで絞り込んで、品種の分布を確認します」。**
