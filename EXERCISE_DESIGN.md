> 対象ファイル：`exercises/ch0*/ch*_answer.ipynb`  
> 更新日：2026-04-14

---

## 1. コース全体の設計思想

### 1-1. 「コードを書く」のではなく「インサイトを言語化する」

このコースの最終ゴールは **「データから何が言えるか」を自分の言葉で説明できること** です。  
コードはその手段に過ぎません。各章の末尾にある「AI 禁止」セクションがコースの核心であり、演習全体は「コードを実行する → グラフ・数値を見る → 言葉にする」という繰り返し構造になっています。

### 1-2. 章間の連鎖構造

各章は独立した演習ではなく、前章の「仮説」が次章の「答え」になる連鎖として設計されています。

```
Ch.1 数値で仮説を立てる（proline, alcohol が品種を分けそう）
  ↓
Ch.2 グラフで仮説を視覚化する（分布・相関を目で確認）
  ↓
Ch.3 機械学習で仮説を検証する（特徴量重要度で予想と照合）
  ↓
Ch.4 回帰を応用して異常を検知する（「正常」からのズレで判断）
  ↓
Ch.5 全手順を新データで再現する（乳がんデータで Ch.1〜3 を通し実行）
```

### 1-3. 難易度の段階設計

各章の設問には `★☆☆` / `★★☆` / `★★★` の難易度タグがついています。

| タグ | 位置づけ | 対応方針 |
|------|----------|----------|
| ★☆☆ | Copilot への質問例がそのまま使えるレベル | 全員が完走するよう設計 |
| ★★☆ | プロンプトを自分でカスタマイズする必要があるレベル | 講師が巡回サポート |
| ★★★（発展） | 時間に余裕のある受講生向け | スキップ可。解説は口頭のみ |

### Ch.1　`ch1_data_analysis_answer.ipynb`
**テーマ：データを「読む」── 数値で現実を掴む**

#### 学習目標
- DataFrame の基本操作（`.shape`, `.head()`, `.info()`, `.describe()`）を習得する
- 「欠損値の確認」「ばらつきの大きい列の発見」など、**分析前の必須確認手順**を身につける
- `groupby` で集計し、品種間の数値差を定量的に捉える
- 「数値の差 → 言語化 → 仮説」という思考ルーティンを最初に体験する

#### 演習の流れと各ステップの意図

| ステップ | 内容 | 意図 |
|----------|------|------|
| **STEP 0** | `df.shape` でデータの「大きさ」を確認 | 最初に全体像を把握する習慣づくり |
| **Q1-1** | `df.head()` で先頭行を表示 | 列名・値の型を目で確認する |
| **Q1-1追加** | `df.tail()` で末尾行も確認 | データの「端」が異常でないかチェックする視点 |
| **Q1-2** | `df.info()` で型・欠損を一覧確認 | 「欠損ゼロ確認」がクリーンデータの前提であると理解させる |
| **Q1-2補足** | `df.isnull().sum()` で列ごとに欠損数を確認 | info の読み方の補強 |
| **Q1-3** | `df.describe()` で統計量全体を確認 | mean / std / min / max の意味を実データで体感する |
| **Q1-3追加** | std ランキングで `proline` が突出していることを発見 | **Ch.2・Ch.3 への伏線**：重要変数に先に気づかせる |
| **STEP 1 チェック（AI禁止）** | 品種数・欠損有無・std 最大列を記入 | 「見た」ではなく「わかった」に変換させる |
| **問1** | `unique()` / `value_counts()` でカテゴリ確認 | ターゲット変数（品種）の分布を把握する |
| **問2** | `groupby` で品種別平均を比較 | 「品種によって数値が違う」＝機械学習が成立する前提を体感 |
| **問3（AI禁止）** | 気づきを文章で記入 | **最重要**：インサイトを言語化する最初の経験 |
| **問4（発展）** | 条件フィルタリングで「高アルコール×高プロリン」の品種を確認 | 仮説（問3）を実際に検証するクロージング体験 |

#### 章末に学べていてほしいこと
> 「proline の std が他の列の10倍以上ある → 品種間でばらつきが大きい変数 → 品種を分けるカギになりそう」という仮説が自分の言葉で言える状態

### Ch.3　`ch3_classification_answer.ipynb`
**テーマ：機械学習で品種を分類する ── ルールを「書く」から「学ばせる」へ**

#### 学習目標
- X（特徴量）と y（ターゲット）という機械学習の基本的なデータ構造を理解する
- `train_test_split` でデータを訓練・テスト用に分割する意義を理解する
- `RandomForestClassifier` を使って「学習→予測→評価」の一連の流れを体験する
- 正解率・混同行列・特徴量重要度の3段階評価で「どのくらい正確か」以上の情報を読み取る
- Ch.2 の仮説（重要変数の予想）と Ch.3 の特徴量重要度を照合する

#### 演習の流れと各ステップの意図

| ステップ | 内容 | 意図 |
|----------|------|------|
| **STEP 1（Q1-1）** | `df.drop()` で X を作る | 「ターゲット列を除いた残りが特徴量」という分離の概念を体験 |
| **STEP 1（Q1-1追加）** | X と y の形（`.shape`）を確認 | X が 178×13、y が 178 であることで概念を数値で確認 |
| **Q1-2** | `train_test_split` でデータ分割 | 訓練とテストを分ける理由（公平な評価）を理解する |
| **Q1-2追加** | 分割後の件数・クラス分布を確認 | 分割がランダムでもバランスが保たれることを確認 |
| **問1** | `model.fit()` で学習 | 「学習」が何をしているかを概念レベルで把握 |
| **問1-b追加** | `n_estimators` / `n_features_in_` を確認 | モデル設定が意図通りかを確認する習慣づけ |
| **問1-c追加** | 1件だけ取り出して予測 | `predict` の出力が「品種名」であることを体感させる |
| **問2** | `accuracy_score` で正解率を計算 | 最も基本的な評価指標を習得 |
| **問2-b追加** | 訓練精度 vs テスト精度を比較 | 「過学習」の概念を実際の数値で体感 |
| **問3** | `confusion_matrix` + ヒートマップ | 「どのクラスを間違えたか」を視覚化する最重要ステップ |
| **Q3-1追加** | 混同行列を品種名付き DataFrame で表示 | 行列の読み方を実際のラベルと紐づけて理解 |
| **Q3-2** | `feature_importances_` を棒グラフで表示 | Ch.2 の仮説（proline・alcohol が重要）を検証する |
| **問3まとめ追加** | `classification_report` で全指標を一覧確認 | Precision / Recall / F1 の意味を実務的に理解 |
| **STEP 3 考察（AI禁止）** | 仮説検証の結果と「正解率97%は良いか」を記入 | **最重要**：指標の数値を「意味」に変換する練習 |
| **問4（発展）** | 品種ごとに Precision / Recall を個別計算 | 評価指標の詳細解釈（時間余裕のある受講生向け） |

#### 章末に学べていてほしいこと
> 「97%の正解率は高い。でも混同行列を見ると wine_1 と wine_2 の混同が残っており、用途によっては不十分かもしれない。Precision と Recall のどちらを優先すべきかはビジネス文脈による」という多面的評価の視点を持てる状態

### Ch.4　`ch4_anomaly_answer.ipynb`
**テーマ：時系列データの異常を検知する ── 「正常」を学ばせて「異常」を見つける**

#### 学習目標
- 「異常ラベルなしで異常を検知する」という実務的な課題設定を理解する
- `shift()` で時系列の過去値を特徴量化する（ラグ特徴量）操作を習得する
- 「正常期間だけでモデルを学習させる」という異常検知の設計思想を理解する
- `LinearRegression` による予測と残差（スコア）の関係を把握する
- パーセンタイルでしきい値を設定し、異常フラグを作る手順を体験する

#### 演習の流れと各ステップの意図

| ステップ | 内容 | 意図 |
|----------|------|------|
| **STEP 1** | 折れ線グラフでデータを目視確認 | time=80〜89 で異常（振れ幅増大）が目視できることを確認 |
| **STEP 1追加** | `describe()` で基本統計量を確認 | 数値とグラフを往復する習慣を維持 |
| **STEP 2** | `shift(1)` で `lag_1` 列を追加 | 「1ステップ前の値を特徴量にする」時系列特有の前処理 |
| **STEP 2追加** | `head()` で lag_1 の中身を確認 | NaN の発生と dropna の必要性を視覚的に理解 |
| **問1** | 正常期間（time < 80）だけで `LinearRegression` 学習 | 「正常パターンだけ学習させる」異常検知の核心設計 |
| **問1-a追加** | 正常期間の件数を確認 | 学習データの規模感を把握する習慣 |
| **問1追加** | モデルの傾き・切片を確認 | 線形回帰が「直線の式」であることを実数値で体感 |
| **問2** | 全期間で予測し `residual`（残差）と `score`（絶対値）を計算 | 「予測と現実のズレ」が異常スコアになる仕組みを理解 |
| **問2-3追加** | 正常期間 vs 異常期間のスコア統計を比較 | 異常期間でスコアが有意に大きいことを数値で確認 |
| **問3** | 95パーセンタイルをしきい値として異常フラグを作成 | しきい値設定の「正常データの外れ値を基準にする」考え方 |
| **問3-1追加②** | 異常行だけを抽出して確認 | 検知結果を実際のデータと照合する検証ステップ |
| **Q3-2** | 正常（青）・異常（赤）を色分けして折れ線グラフを描画 | 「視覚的にわかりやすい出力」を作る最終ゴール |
| **STEP 3 振り返り（AI禁止）** | 仕組みの説明と実務応用を記入 | **最重要**：異常検知の概念を自分の言葉で説明できるか確認 |
| **問4（発展）** | しきい値（パーセンタイル）を 90/95/99 で変えて比較 | 感度とノイズのトレードオフを体感（時間余裕のある受講生向け） |

#### 章末に学べていてほしいこと
> 「正常期間のデータで学習した予測モデルと実際の値のズレ（残差の絶対値）をスコアとし、正常期間スコアの95パーセンタイルをしきい値にする。異常ラベルがなくても機械的に検知できる」という仕組みを自分の言葉で説明できる状態

### Ch.5　`ch5_comprehensive_answer.ipynb`
**テーマ：総合ミニ演習 ── Ch.1〜3 の手順を新しいデータで再現する**

#### 学習目標
- Ch.1〜3 で習得した全手順を、新しいデータセット（乳がんデータ）に自力で適用できる
- 「不均衡データ（良性 63% / 悪性 37%）」という実務的な課題に対し、正解率だけで評価しない視点を持つ
- Precision（偽陽性のコスト）と Recall（見逃しのコスト）のトレードオフを医療文脈で理解する
- コース全体の学習を「データ分析の共通フロー」として言語化・定着させる

#### 演習の流れと各ステップの意図

| ステップ | 内容 | 意図 |
|----------|------|------|
| **Step 1** | `load_breast_cancer()` から DataFrame を作成 | Ch.1 の `load_wine` と同じ手順で新データを扱う体験 |
| **Q1-1補足** | `df.info()` で型・欠損を確認 | Ch.1 の必須確認手順を新データで反射的に実施させる |
| **Q1-2** | 件数・基本統計の確認 | データの規模感と目標変数の分布（不均衡）を早期に把握 |
| **Q1-2追加** | クラスバランスを割合で確認 | 「良性62%・悪性37%」という不均衡を数値で意識させる |
| **Step 2** | ヒストグラム・箱ひげ図・散布図・ヒートマップ | Ch.2 の可視化手順を新データで再現し、仮説を立てる |
| **Q2-2追加** | 散布図を診断結果で色分け | Ch.2 の品種別散布図と同じ発想の応用 |
| **Step 3** | X/y 分離 → `train_test_split` → `RandomForest` 学習 | Ch.3 の STEP 1〜問1 を圧縮して再現 |
| **Q3-3追加** | 1件だけ取り出して予測 | Ch.3 の問1-c の応用。予測出力を実感させる |
| **Step 4（Q4-1）** | 正解率の計算と訓練/テスト精度の比較 | 過学習の有無を確認する Ch.3 の手順を定着 |
| **Q4-2** | 混同行列 + 表形式で表示 | **最重要**：悪性の見逃し（FN）を直視させる |
| **Q4-3** | 特徴量重要度（上位10件）を棒グラフで表示 | 「どの計測値が診断に重要か」という医学的知見との対照 |
| **発展** | `classification_report` で Precision / Recall / F1 を一覧確認 | Ch.3 問3まとめの反復。不均衡データで Recall の重要性を再認識 |
| **考察（AI禁止）** | 重要変数・正解率96%の実務的評価・全体フローの言語化 | **最重要**：コース全体の総括。「データ分析の共通フロー」として定着させる |

#### 章末に学べていてほしいこと
> 「最も重要な変数・正解率96%でも悪性の見逃しが残る → 医療では Recall 最優先 → しきい値調整が必要」という総合判断ができ、「データ確認 → 可視化・仮説 → モデル学習 → 評価 → 言語化」という全体フローを自分の言葉で説明できる状態

## 3. 章間の学習連鎖まとめ

### 「仮説 → 検証」の串刺し構造

```
Ch.1  proline の std が突出していると気づく（数値）
  ↓ 仮説「品種を分ける重要変数では？」
Ch.2  proline の箱ひげ図・散布図で品種の分離を視覚確認（グラフ）
  ↓ 仮説「機械学習モデルでも proline が上位に来るはず」
Ch.3  特徴量重要度で proline が上位に来ることを確認（検証完了）
  ↓ 回帰という新しい手法を応用
Ch.4  LinearRegression を「異常検知」として使う発展的応用
  ↓ 全手順を新データで再現
Ch.5  乳がんデータで Ch.1〜3 の全手順を通し実行し、Recall の重要性を体得
```

### 各章が使い回す概念の累積

| 概念 | Ch.1 | Ch.2 | Ch.3 | Ch.4 | Ch.5 |
|------|:----:|:----:|:----:|:----:|:----:|
| DataFrame の基本操作 | ◎ | ○ | ○ | ○ | ○ |
| groupby / value_counts | ◎ | ○ | - | - | ○ |
| グラフ（分布・比較） | - | ◎ | ○ | ◎ | ○ |
| X / y の分離 | - | - | ◎ | ◎ | ○ |
| train_test_split | - | - | ◎ | - | ○ |
| 学習・予測（fit/predict） | - | - | ◎ | ◎ | ○ |
| 評価（正解率・混同行列） | - | - | ◎ | - | ○ |
| Precision / Recall / F1 | - | - | ○ | - | ◎ |
| 残差・しきい値 | - | - | - | ◎ | - |

◎ 初登場・中心概念　○ 定着・応用　- 不使用

## 4. 「AI 禁止」セクションの設計意図

各章に最低 1〜2 か所ある「AI 禁止」セクションは、演習の核心です。

### 配置の原則
- **STEP の区切り目**に設置 → コードを動かした直後の状態で言語化させる
- **章末の総まとめ**として設置 → 章全体を通して何がわかったかを整理させる

### 問い方の設計
| 設問パターン | 目的 | 対象章 |
|-------------|------|--------|
| 「〇〇の結果から何がわかりましたか？」 | 観察から気づきへの変換 | Ch.1〜4 の気づきメモ |
| 「仮説と一致していましたか？」 | 仮説検証サイクルの意識化 | Ch.3 STEP 3 |
| 「このモデルは実務で使えますか？」 | 指標を「意味」に変換する批判的思考 | Ch.3 STEP 3、Ch.5 考察 |

## 5. アンサーファイルの運用方針

| 項目 | 方針 |
|------|------|
| **使用タイミング** | 演習終了後に講師が画面共有して解説。配布は研修後 |
| **AI 禁止セクションの模範解答** | アンサーファイルに記載済み。受講生の回答と照らし合わせる材料として使用 |
| **発展問題（★★★）** | 時間内に解けなかった受講生へ研修後の自習用として案内 |
| **コードコメントの水準** | 1ブロック1コメント。行ごとの詳細説明は口頭説明で補う |
| **中間出力（print 等）** | 最終成果物の出力のみ残す。中間確認は講師が口頭で説明 |
