# Ch.4　時系列データと異常検知

### ── 「同じ道具」で異常を見つける ──

**本章のゴール**
- Ch.3 と同じ `fit → predict` の型で、センサーデータの異常点を検出できる

```
【Ch.4 でやること】

時系列データを折れ線グラフで目視確認
          ↓
「1つ前の値（lag_1）」を入力として学習させる（LinearRegression）
          ↓
「予測が大きく外れた点」＝ 異常スコアが高い点
          ↓
しきい値を設定 → 異常フラグをつける → グラフで確認
```

---

## 実務での活用イメージ

```
【時系列の異常検知が使われている場面】

製造設備の保守
  センサーの振動・温度（時刻順）→ 通常範囲を超えた → 故障の前兆

金融の不正検知
  取引金額の時系列データ → 急激な高額取引 → 不正の可能性

インフラ監視
  サーバーの CPU・メモリ使用率 → 急上昇 → 障害の前兆

小売の売上管理
  日別・週別の売上データ → 急落・急騰 → キャンペーン効果・トラブル
```

> 📌 **「いつもと違う動き」を自動で検出するのが異常検知の目的です。**  
> 人が全データを目視確認するのは不可能。モデルが常時監視してくれます。

---

## 時系列データとは ── 行の「順番」に意味がある

```
【通常のデータ（Ch.1〜3）】    【時系列データ（Ch.4）】

  alcohol  proline  品種            時刻    センサー値
    14.2    1065    wine_0            0      32.1
    13.2    1050    wine_0    →       1      33.4
    12.4     500    wine_1            2      31.8
    13.8     900    wine_2            3      62.5  ←  突然跳ね上がった！
                                      4      32.0

行の順番に意味はない             行の順番が重要！
どの順番でも分析できる           過去の値が未来の値の「ヒント」になる
```

### なぜ順番が大事か

> 📌 **「62.5 という値が 1点だけ」では異常かどうかわかりません。**  
> 「前後が 31〜33 の範囲なのに、この 1点だけ 62.5」という「文脈」があるから異常とわかります。

## Ch.3 との接続 ── 「道具は同じ」

```
【Ch.3 と Ch.4 の比較】

                Ch.3（分類）              Ch.4（異常検知）
                ────────────────          ───────────────────
X（入力）    化学成分 13列               1つ前の値（lag_1）
y（正解）    品種名（カテゴリ）           次の時刻の値（数値）
モデル       RandomForest               LinearRegression
学習         fit(X, y)                  ── 全く同じ！ ──
推論         predict(X)                 ── 全く同じ！ ──
```

> 📌 **新しいアルゴリズムを覚えるわけではありません。**  
> 変わるのは「X に何を入れるか」と「y が数値になる」だけです。

## 線形回帰の基本 ── 「直線を当てはめる」とは？

```
【線形回帰 ＝ データに「最もよく当てはまる直線」を求めること】

  y = a × x + b

  a（傾き）   ＝ 「x が 1増えると y がどれだけ変わるか」
  b（切片）   ＝ 「x が 0 のときの y の値」

【学習（fit）でやっていること】

  ●    ●                    ●    ●
     ●      →  直線を引く  →   ／  ●
  ●     ●                 ●  ／  ●
                         ／
  データの点が             「最も近い直線」を
  バラバラにある            自動で見つける（これが fit ！）

【予測（predict）でやっていること】

  新しい x（直前の値）を式に代入して y（次の値）を計算する

  例：lag_1 = 32.0
      y = 1.02 × 32.0 + 0.5 ≈ 33.1  ← これが「予測値」
```

> 📌 **Ch.3 の RandomForest も Ch.4 の LinearRegression も、**  
> **「`fit` で学習・`predict` で予測」の型はまったく同じです。**  
> 違いは「直線で予測するか・木の多数決で予測するか」だけです。

---

## LinearRegression（線形回帰）とは

```
【「直線」で次の値を予測する】

次の値（y）
  ↑
70│
60│               ●  ← 異常！直線から大きく外れている
50│
40│
30│   ●
   │      ●   ●          ← 正常なデータは直線の近くに集まる
20│           ●  ●  ●
10│
  └─────────────────────────→ 直前の値（lag_1）
             直線（正常パターン）

正常な点 ＝ 直線の近くにある   → 予測誤差が小さい
異常な点 ＝ 直線から大きく外れる → 予測誤差が大きい = 異常スコア大
```

> 📌 **「直線から大きく外れた点 ＝ 正常パターンから逸脱した点 ＝ 異常の疑い」**

---

## ラグ特徴量 ── 「1つ前の値」を変数にする

```
【ラグ特徴量（lag_1）の作り方】

元の時系列データ：              ラグ特徴量を作った後：
  時刻  値                        時刻   lag_1（X）   値（y）
   0   32.1                         1      32.1        33.4  ← 32.1の次は33.4
   1   33.4     →                   2      33.4        31.8  ← 33.4の次は31.8
   2   31.8                         3      31.8        62.5  ← 31.8の次は62.5（異常！）
   3   62.5                         4      62.5        32.0
   4   32.0                                ↑
                                      1つ前の値
```

### なぜ「1つ前」を使うのか

```
センサー値の特徴：「前の値に近い値が続く」（32→33→31→33 のように緩やかに変動）
          ↓
「直前の値が 31.8 なら次も 31〜34 のはず」という直線的な関係を学習できる
          ↓
62.5 が来たとき → 「31〜34 のはずなのに大きく外れている！」→ 異常の疑い
```

---

## なぜ「正常期間だけ」で学習するのか

```
【正しい方法：正常期間だけで学習する】

  正常期間（time 0〜79）だけで fit()
          ↓
  「値が前後 ±2〜3 程度の範囲に収まる」という正常パターンを覚える
          ↓
  全期間で predict() → 異常スコアを計算
          ↓
  time 80〜90 のスパイク → 誤差が大きい → 「いつもと違う！」と検出できる

【問題のある方法：異常を含む全期間で学習する】

  全期間（time 0〜119）で fit() ← スパイクも含む
          ↓
  「スパイクも正常パターンの一部」として覚えてしまう
          ↓
  異常が検出できなくなる！
```

> 📌 **異常検知では「正常なデータだけで学習させる」ことが最大のポイントです。**

## 予測誤差 ＝ 異常スコア

```
【予測誤差で異常を見つける仕組み】

学習済みモデル（正常パターンを覚えている）に全期間を予測させる

正常な時刻（time 0〜79 や time 90〜119）：
  実際の値 ≈ 予測値  →  誤差 ≈ 0〜3  →  異常スコア 小

異常な時刻（time 80〜90 のスパイク）：
  実際の値 62.5 − 予測値 32.0 → 誤差 30.5 → 異常スコア 大 ← ここが異常！

【異常スコアの時系列グラフ】

スコア
35│                  ████   ← ここが高い！（time 80〜90）
30│                ██████
25│                ████████
20│
10│
 5│████████████████████████████████
 0└────────────────────────────────→ 時刻
   0          40        80       120
```

## しきい値 ── 「どのくらい外れたら異常か」

```
【しきい値の設定方法】

正常期間（time 0〜79）の異常スコアの分布
      │
      │  上位 5%（＝最もスコアが高い 5%）の値をしきい値とする
      │
      ▼
  ─────────────────────────────────┬──── ← しきい値（例：8.5）
  ここは正常（スコアが低い 95%）   │ここは異常（上位 5%）

【しきい値を変えると何が変わるか】

しきい値を低くする（ゆるく）→ 異常判定が増える → 見逃しは減るが誤検知が増える
しきい値を高くする（厳しく）→ 異常判定が減る  → 誤検知は減るが見逃しが増える

  医療・安全関連（見逃し厳禁）→ しきい値を低く（ゆるく）設定
  コスト重視（誤った停止が損失）→ しきい値を高く（厳しく）設定
```

> 📌 **「見逃し vs 誤検知」のバランスは、ビジネスの文脈で判断します。**

**Q1．製造設備の故障前兆を検知するシステムを作るとします。**  
**「故障を見逃す」と「正常なのに故障と誤検知する」、どちらが業務上の問題が大きいですか？**

> 💡 **考え方のヒント：**  
> ・「故障を見逃す」→ 設備が壊れてラインが止まる。修理コスト・納期遅延・安全事故のリスク  
> ・「正常なのに止める」→ 不必要な点検コスト・稼働率の低下  
> 「どちらの損失が大きいか」はビジネスによって異なります。命や大きなコストに関わる場合は見逃し厳禁です。

---

**Q2．「しきい値を低く（ゆるく）設定する」と、**  
**検出件数・見逃し・誤検知はそれぞれどう変化しますか？**

> 💡 **考え方のヒント：**  
> しきい値を低くすると「ちょっと外れただけでも異常と判定する」ゆるい基準になります。  
> ・検出件数：**増える**（異常と判定される点が増える）  
> ・見逃し：**減る**（微小な異常も拾えるようになる）  
> ・誤検知：**増える**（正常なのに異常と判定される件数が増える）  
> 「見逃し↓ ＝ 誤検知↑」という関係は Ch.3 の Precision / Recall と同じトレードオフです。

## 演習との接続 と まとめ

### 演習でやること

```
numpy で生成した時系列データ（120点・スパイク1箇所：time 80〜90）
  ↓ ① 折れ線グラフで目視確認
  ↓ ② ラグ特徴量（lag_1）を作る
  ↓ ③ 正常期間（time 0〜79）だけで LinearRegression を学習
  ↓ ④ 全期間を予測 → 異常スコア（予測誤差の絶対値）を計算
  ↓ ⑤ 異常スコアの折れ線グラフで分布を確認する
  ↓ ⑥ しきい値（正常期間の上位5%）で異常フラグを設定
  ⑦ グラフに赤い点でハイライト → time 80〜90 に集中すれば成功！
```

### Ch.4 で学んだこと

| 概念 | 内容 |
|------|------|
| **時系列データ** | 行の順番が重要。過去の値が未来の手がかりになる |
| **ラグ特徴量（lag_1）** | 「1つ前の値」を X として追加する |
| **LinearRegression** | 「直前の値 → 次の値」の直線的な関係を学習 |
| **正常期間で学習** | 異常を含まないデータで「正常パターン」を覚えさせる |
| **予測誤差 ＝ 異常スコア** | 正常なら誤差小、異常なら誤差大 |
| **しきい値のトレードオフ** | 見逃し vs 誤検知。ビジネス文脈で判断する |
