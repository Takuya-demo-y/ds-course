### スライド 01-05｜欠損値（NaN）とは

**重要メッセージ**
> ⚠️ 現実のデータには必ず「空白」が存在します。そのままモデルに入れるとエラーになります。

- **NaN = Not a Number**（値が記録されていない）  
  原因：入力ミス・センサー故障・アンケート未回答 など

- **確認方法**：`df.isnull().sum()` で各列の欠損件数を表示

- **欠損率の確認**：`df.isnull().sum() / len(df) * 100`

**コード例**
```python
# 各列の欠損件数を確認
df.isnull().sum()

# 欠損率（%）を確認
(df.isnull().sum() / len(df) * 100).round(1)
```

---

### スライド 01-06｜欠損値の処理方針

| 方法 | コード | 使う場面 | リスク |
|------|--------|---------|--------|
| 削除 | `df.dropna()` | 欠損が少量かつランダムな場合 | データ量が減る |
| 平均値で補完 | `df.fillna(df.mean())` | 数値データで欠損が一定量ある場合 | 分布が変わる可能性 |
| グループ別平均 | `groupby + transform` | グループ間に差がある場合 | コードが複雑 |

**重要ポイント**
> ⚠️ 「どれが正解」は一概に言えない。データの性質とビジネス要件に合わせて判断する

### スライド 01-07｜フィルタリングと集計

**🔍 フィルタリング：条件で絞り込む**

```python
# アルコール度数が 13.5 以上
high_alc = df[df["alcohol"] >= 13.5]

# 複数条件（& で AND）
filtered = df[(df["alcohol"] >= 13.5) & (df["color_intensity"] >= 5.0)]
```

**📊 groupby：グループごとに集計する**

```python
# 品種ごとの平均を集計
df.groupby("品種")[["alcohol", "proline"]].mean().round(2)

# ソート
df.sort_values("proline", ascending=False).head()
```

**実務例**
> 「都道府県ごと」「月ごと」の売上平均集計が groupby 一発で出来る

### スライド 02-04｜ヒストグラム：データの分布を見る

**読み方のポイント**
- 横軸：値の範囲 ／ 縦軸：その範囲に含まれるデータ件数
- 山が左に偏っている → 低い値が多い
- 2つの山がある → 2つのグループが混在している可能性
- 品種別に色分けすることで「品種ごとの違い」がわかる

**コード例**
```python
# 品種ごとのヒストグラム
for name, group in df.groupby("品種"):
    plt.hist(group["alcohol"], bins=15, alpha=0.6, label=name)
plt.legend()
plt.show()
```

**観察の例**
> 🔍 Barolo は他の品種よりアルコール度数が高い傾向がある

### スライド 02-05｜箱ひげ図：グループ間の分布を比較する

**箱ひげ図の各要素の意味**

### スライド 02-06｜散布図と相関係数

**相関係数 r の読み方**

| 相関係数 r | 意味 |
|-----------|------|
| r ≈ +1 | 強い正の相関（一方が増えると他方も増える） |
| r ≈ 0 | 相関なし（2変数に関係がない） |
| r ≈ −1 | 強い負の相関（一方が増えると他方は減る） |

**コード例**
```python
# 相関行列とヒートマップ
corr = df.corr()
sns.heatmap(corr, annot=True, fmt=".2f", cmap="coolwarm", center=0)
```

**重要な注意点**
> ⚠️ 相関 ≠ 因果関係  
> 例）アイスが売れると溺死者が増える → 原因は「夏（気温）」

**実務活用**
> 💡 相関が高い変数 = 予測に使える候補（ただし多重共線性に注意）

### スライド 03-04｜グレースケールとカラー（RGB）の違い

| 項目 | 🔲 グレースケール | 🌈 カラー（RGB） |

# カラー画像をグレースケールに変換

### スライド 03-07｜画像データの統計分析

**キーメッセージ**
> 画像 ＝ 数値 → 統計量を計算できる → 機械学習の特徴量として使える！

**主な統計量と意味**
- **平均ピクセル値** → 画像の「明るさ」を数値化
- **標準偏差** → 明暗のばらつき（コントラスト）
- **最大・最小** → 最も明るい/暗い箇所
- **0超のピクセル数** → 「描かれている量」

**コード例**
```python
arr = np.array(img_large)

print("明るさ（平均）:", arr.mean())
print("コントラスト:", arr.std())
print("描画率:", (arr > 0).mean() * 100, "%")
```

### スライド 04-04｜なぜスケーリングが必要か

**問題の具体例**
> ⚠️ アルコール度数（約12〜15）とプロリン含有量（約278〜1680）→ スケールが全く違う！  
> → 大きい値の変数が不当に影響力を持ってしまう

**解決策：StandardScaler**
> 各変数を **平均0・標準偏差1** になるよう変換 → すべての変数のスケールが揃う

**コード例（最重要ルール付き）**
```python
scaler = StandardScaler()

# fit は訓練データのみ！
X_train_scaled = scaler.fit_transform(X_train)

# テストデータは transform のみ
X_test_scaled = scaler.transform(X_test)

# ❌ 絶対NG: scaler.fit_transform(X_test) ← データリーク！
```

**🔑 最重要ルール**
> `scaler.fit()` は訓練データのみ！テストデータには `transform()` だけを使う（データリーク防止）

### スライド 04-06｜混同行列（Confusion Matrix）の読み方

**混同行列の4つのマス**

|  | 予測: 陽性 | 予測: 陰性 |
|--|-----------|-----------|
| 実際: 陽性 | **TP（正解）** ✅ 陽性→陽性と予測 | **FN（見逃し）** ⚠️ 陽性→陰性と予測 |
| 実際: 陰性 | **FP（誤検知）** ⚠️ 陰性→陽性と予測 | **TN（正解）** ✅ 陰性→陰性と予測 |

**評価指標**

| 指標 | 計算式 | 重視する場面 |
|------|--------|------------|
| Accuracy（正解率） | `(TP+TN) / 全体` | 全般的な正確さ |
| Precision（適合率） | `TP / (TP+FP)` | 誤検知を減らしたい場合 |
| Recall（再現率） | `TP / (TP+FN)` | 見逃しを防ぎたい場合 |

**補足**
> 対角成分（TP + TN）= 正解した件数　対角以外 = 誤分類 → どう間違えたかが重要

---

### スライド 04-07｜正解率（Accuracy）だけでは不十分な場合

**具体例：1,000件中 990件が「陰性」、10件が「陽性」のデータ**
```
「全部陰性」と予測するモデル
→ 正解率 = 99%（高い！）
→ 陽性10件を全部見逃している！
→ Recall = 0%（使えないモデル）
```

**業務シーン別の重視指標**

| 業務シーン | 重視する指標 | 理由 |
|-----------|------------|------|
| 医療検査（がん検診） | Recall（再現率） | 陽性の見逃しが命取り（FN を減らす） |
| 迷惑メールフィルタ | Precision（適合率） | 正常メールを誤削除したくない（FP を減らす） |
| 不良品検査 | Recall（再現率） | 不良品の見逃しはクレームに直結 |

**まとめ**
> 🔑 正解率は必ず見る指標だが、混同行列と合わせて評価することが重要
