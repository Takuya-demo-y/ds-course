## 指摘事項① 画像処理チャプターの見直し

### 元の課題

| 項目 | 内容 |
|------|------|
| 現状案 | Pillow を用いて画像の読み込み・色変換・フィルタ処理（ぼかし、シャープネス）を学ぶ内容 |
| 問題点 | ピクセルの明るさを変えるだけでは、受講生が実務での活用イメージを持ちにくい |
| 要望 | 機械学習モデルと組み合わせた「異常検知」など、実務に即したアウトプットを示すべき |
| 制約 | PCスペック（JupyterLab 環境）と時間制約があるため、軽量なライブラリ（scikit-learn 等）を優先する |

### 対応内容（✅ 対応済）

**Ch.3「画像データと異常検知」として全面改訂。**

| 変更前 | 変更後 |
|--------|--------|
| Pillow による画像フィルタ操作 | IsolationForest（scikit-learn）による異常検知 |
| 「ぼかし・シャープネス」などの画像編集 | 「正常パターンからの逸脱を検出する」実務直結のアウトプット |
| 画像処理の操作知識 | 「画像 ＝ 数値の配列」という本質的な概念理解 |

**座学・演習の役割分担：**

```
座学（30分）: 2D 散布図を使って「密集 = 正常 / 孤立 = 異常」を目で理解
              → IsolationForest のアイデアを知識ゼロでも直感的に理解できる

演習（40分）: 工場温度センサーマップ（ヒートマップ）を操作
              → 「色 = 数値の配列 = 画像」を手を動かして体験
```

**使用ライブラリ：** `numpy`・`matplotlib`・`scikit-learn 0.23.2`（社内 PC 制約内）  
**Pillow は本コースで使用しない。**

## 指摘事項② 講義と演習の構成

### 元の課題

| 項目 | 内容 |
|------|------|
| 時間配分 | 座学20分（ハンズオン形式）＋ 演習40分 |
| 問題点① | 初心者は「わからないコード」が増えるだけで認知負荷が高まり脱落する |
| 問題点② | 「何ができるか」を重視し、不要なコードオプションは削いでメソッドを簡略化する必要がある |
| 問題点③ | 生成AIを活用させる場合でも、前提となる基礎知識をどう定着させるかが課題 |

### 対応内容（✅ 対応済 ／ ⚠️ 一部変更あり）

#### 座学の時間：20分 → **30分に延長**（変更理由あり）

元の指摘では座学20分となっていたが、以下の理由から**30分に変更**している。

```
理由①  初学者が概念を定着させるには 20 分では不足する
理由②  ハンズオンデモ（講師がコードを実行しながら説明する時間）を含めると
        20 分では概念説明と実演の両立が困難
理由③  10 分延長した分は Ch.3 を昼またぎにする設計で全体の時間を調整済み
```

タイムラインへの影響：

```

9:35   Ch.1  座学30分 + 演習40分    70分
10:45  休憩                         10分
10:55  Ch.2  座学30分 + 演習40分    70分
12:05  Ch.3  座学 前半（概念導入）   10分  ← 昼食前のフック
12:15  昼休憩                       60分
13:15  Ch.3  座学 後半（主要解説）   20分  ← 昼食後に本題（合計30分）

```

#### 認知負荷への配慮（対応済）

| 指摘事項 | 対応内容 |
|---------|---------|
| コードを削ぎ落とす | **座学スライドにコード説明を一切載せない**。概念・アナロジー・発問のみ |
| 「何ができるか」を重視 | 各チャプターの冒頭で「このチャプターで作るもの」を先に提示 |
| ハンズオン形式 | 各スライドの最後（08枚目）に「コードで確認する（デモ）」を設け、講師が実行しながら説明 |

#### 生成AI活用の基礎知識定着（対応済）

```
フロー：座学で概念を理解 → 演習でCopilot Chatを使ってコードを生成 → 結果を自分で解釈

ルール：
  ・Copilot Chat（ウェブブラウザ版）のみ使用

  ・座学中は使用禁止（まず概念を頭に入れる）
  ・演習では決まったテンプレートで質問する

テンプレート（全演習共通）：

  【データの形】DataFrame で列名は〇〇、行数は〇〇件
  【環境】Python 3.8.6、Windows、JupyterLab
  【わからない点】〇〇の書き方 / 〇〇でエラーが出た
```
