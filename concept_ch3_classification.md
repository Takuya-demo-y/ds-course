## Ch.3 前半｜X と y の役割

```
【X と y の関係】

         X（特徴量：手がかり）              y（正解ラベル：答え）
  ┌────────────────────────────┐       ┌──────────┐
  │ alcohol  proline  ...など  │       │  wine_0  │
  │   14.2    1065   ...      │  →    │  wine_0  │
  │   13.2    1050   ...      │       │  wine_1  │
  └────────────────────────────┘       └──────────┘
      モデルへの「入力」（13列）          モデルが予測する「答え」
```

```
⛔ 重要なルール：y（品種名）は X に含めてはいけない

  なぜ？ → 答えを見せながら学習させると「カンニング」と同じになる
            → 見たことのあるデータは正解できるが
              見たことのないデータでは全く役に立たない
```

> 📌 **「X ＝ 手がかり、y ＝ 正解」この分け方が機械学習の出発点です。**

## Ch.3 前半｜fit() が何をしているか

```
【学習（fit）＝ 試験勉強のようなもの】

人間の試験勉強：
  問題（X）と答え（y）のセットを大量に解く
        ↓
  「このパターンならこの答え」という規則を頭に入れる

機械学習の fit()：
  データ（X）と正解ラベル（y）を大量に見せる
        ↓
  「この成分の組み合わせならこの品種」という規則を自動で見つける

【予測（predict）＝ 本番試験のようなもの】

  初めて見るデータ（X）を渡す
        ↓
  学んだ規則を使って品種（y）を予測して返す
```

```
【機械学習の全体フロー】

① X と y を準備する
        ↓
② 訓練・テストに分割する（なぜ分けるかは次のスライドで）
        ↓
③ fit(X_train, y_train)  ← 訓練データで「勉強させる」
        ↓
④ predict(X_test)         ← 未知データで「試験を受けさせる」
        ↓
⑤ 正解率・混同行列で「何割正解したか」を評価する
```

> 📌 **この ③→④→⑤ の流れが Ch.4 でもそのまま使えます。変わるのは「何を予測するか」だけです。**

## Ch.3 前半｜なぜデータを分割するか ＆ 過学習

```
【データを分割しないと起きる問題（過学習）】

         全データで学習 → 全データでテスト
                  ↓
    モデルが「見たことあるデータ」を暗記している状態
                  ↓
 訓練データ：99% 正解（暗記しているから当然）
 初めて見るデータ：60% しか正解できない（実務では使えない！）

【正しいデータの使い方（Hold-out法）】

 全データ（178件）
    │
    ├── 訓練データ（80% ≒ 142件）  → fit() で「勉強させる」
    └── テストデータ（20% ≒ 36件） → 学習後に1回だけ「試験させる」
                                       ↑
                                「本番試験」なので学習には絶対に使わない
```

> 📌 **「訓練で高精度・テストで低精度」＝ 過学習。暗記した人が本番で失敗するのと同じです。**

## Ch.3 後半｜決定木 と ランダムフォレスト

```
【決定木：Yes/No の質問を繰り返して分類する】

    アルコール度数 > 13.0 ？
          /            \
        Yes              No
         ↓               ↓
  プロリン > 900 ？    → 品種 1 / 2 の可能性
      /       \
    Yes         No
     ↓           ↓
  品種 0       品種 1

長所：なぜこの品種か、理由が説明しやすい
短所：木が深くなると「訓練データを丸暗記」しやすい（過学習）
```

```
【ランダムフォレスト：決定木を「100本作って多数決」する】

データの一部をランダムに選ぶ（100回繰り返す）
         │
   ┌─────┼─────┐
 木①   木②   木③  …（計100本）
 品種0   品種1  品種0
         │
      多数決 → 品種 0（2票） ← 最終予測

長所：精度が高い・過学習しにくい・特徴量重要度がわかる
短所：100本の総合判断なので「なぜこの予測か」の説明が難しい
```

> 📌 **決定木の弱点（過学習しやすい）を「大勢で多数決」することで補ったのがランダムフォレストです。**

## Ch.3 後半｜モデルの評価指標

```
【Accuracy（正解率）：全体で何割正解したか】

テストデータ（36件）を予測させる
         ↓
正解した件数 ÷ 全件数 ＝ 正解率
例：34件正解 / 36件 ＝ 94.4%  ← 今日のゴール：90%以上

目安：
  50% 程度 → ランダムと変わらない（意味なし）
  70〜80% → 業種・用途によっては使えるレベル
  90% 以上 → 今日のゴール
  99%（で訓練も99%）→ 過学習の疑い
```

```
【混同行列：「何を間違えたか」をクラス別に見る】

               予測：品種0  予測：品種1  予測：品種2
実際：品種0  [  ✅ 13   ] [   ❌ 0   ] [   ❌ 0   ]
実際：品種1  [  ❌  0   ] [  ✅ 20   ] [   ❌ 1   ] ← 品種1を品種2と間違えた（1件）
実際：品種2  [  ❌  0   ] [   ❌ 1   ] [  ✅ 12   ] ← 品種2を品種1と間違えた（1件）

対角線（✅）＝ 正解      対角線以外（❌）＝ 間違い

正解率だけでは「どのクラスを間違えやすいか」がわからない
→ 混同行列で「どこで・何を間違えているか」を必ず確認する
```

## Ch.3｜まとめ

| 概念 | 内容 |
|------|------|
| **機械学習** | データからルールを学んで未知データを予測する技術 |
| **X と y** | X = 手がかり（特徴量）、y = 正解ラベル。y は X に含めない |
| **fit() ＝ 勉強** | X と y のセットを大量に見せて「規則」を自動で見つけさせる |
| **predict() ＝ 試験** | 初めて見るデータを渡して予測させる |
| **データ分割（80:20）** | 訓練（勉強）とテスト（本番試験）を分ける。丸暗記防止 |
| **過学習** | 訓練データを暗記し、未知データで精度が落ちる現象 |
| **決定木** | Yes/No の分岐で分類。過学習しやすい |
| **RandomForest** | 決定木を100本作って多数決。精度が高く過学習しにくい |
| **Accuracy + 混同行列** | 正解率（全体成績）と「どこで失点したか」の明細をセットで確認 |
| **Precision / Recall** | Ch.5 で体験してから学ぶ |
