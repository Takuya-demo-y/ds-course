なぜスケーリングが必要か

⚠️ アルコール度数（約12〜15）とプロリン含有量（約278〜1680）
→ スケールが全く違う！ → 大きい値の変数が不当に影響力を持つ
StandardScaler の仕組み
各変数を 平均0・標準偏差1 になるよう変換
→ すべての変数のスケールが揃う
🔑 最重要ルール
scaler.fit() は訓練データのみ
テストデータには transform() だけ！ 
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()

# fit は訓練データのみ！
X_train_scaled = scaler.fit_transform(X_train)

# テストデータは transform のみ
X_test_scaled = scaler.transform(X_test)

# ❌ 絶対にやってはいけない：
# X_test_scaled = scaler.fit_transform(X_test)
# ← テストデータの情報でfitするとデータリーク！

混同行列（Confusion Matrix）の読み方

予測: 陽性
予測: 陰性
実際: 陽性
TP（正解）
陽性→陽性と予測
✅
FN（見逃し）
陽性→陰性と予測
⚠️
実際: 陰性
FP（誤検知）
陰性→陽性と予測
⚠️
TN（正解）
陰性→陰性と予測
✅
重要な評価指標
Accuracy全体の正解率 ＝ (TP+TN) / 全体
Precision予測が正解だった割合 ＝ TP / (TP+FP)
誤検知を減らしたい場合に重視
Recall実際の陽性を拾えた割合 ＝ TP / (TP+FN)
見逃しを防ぎたい場合に重視
対角成分（TP + TN）= 正解した件数
対角以外 = 誤分類 → どう間違えたかが重要

正解率（Accuracy）だけでは不十分な場合

📊 例：1,000件中 990件が「陰性」、10件が「陽性」のデータ
「全部陰性」と
予測するモデル
→
正解率 = 99%
（高い！）
→
陽性10件を
全部見逃している！
→
Recall = 0%
使えないモデル
業務シーン重視する指標理由
医療検査（がん検診）Recall（再現率）陽性の見逃しが命取り（FN を減らす）
迷惑メールフィルタPrecision（適合率）正常メールを誤削除したくない（FP を減らす）
不良品検査Recall（再現率）不良品の見逃しはクレームに直結
🔑 正解率は「必ず見る指標」だが、混同行列と合わせて評価することが重要

特徴量重要度：何が予測に効いているか

特徴量重要度とは
  • ランダムフォレストが自動的に計算する
  • どの特徴量が予測に貢献しているか
  • 値の合計 = 1.0
  • 大きいほど「予測に効いている」
💡 Ch.2のEDAで「プロリンが重要そう」と仮説を立てた
→ 特徴量重要度で仮説を検証できる！ 
import pandas as pd

# 特徴量重要度を取得
importances = pd.Series(
    model.feature_importances_,
    index=wine.feature_names
)

# 重要度の高い順にソート
importances = importances.sort_values(
    ascending=True
)

# 棒グラフで可視化
importances.plot(kind='barh')
plt.title('特徴量重要度')
plt.show()
