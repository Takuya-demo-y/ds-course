Ch.1 データ読み込み・集計

欠損値（NaN）とは

⚠️ 現実のデータには必ず「空白」が存在します
そのままモデルに入れると エラーになります
  • NaN = Not a Number（値が記録されていない） 原因：入力ミス・センサー故障・アンケート未回答 など
  • 確認方法：df.isnull().sum() で各列の欠損件数を表示
  • 欠損率の確認：df.isnull().sum() / len(df) * 100
# 各列の欠損件数を確認
df.isnull().sum()

# 欠損率（%）を確認
(df.isnull().sum() / len(df) * 100).round(1)
Ch.1 データ読み込み・集計

欠損値の処理方針

方法コード使う場面リスク
削除 df.dropna() 欠損が少量かつランダムな場合 データ量が減る
平均値で補完 df.fillna(df.mean()) 数値データで欠損が一定量ある場合 分布が変わる可能性
グループ別平均で補完 groupby + transform グループ間に差がある場合 コードが複雑
⚠️ 「どれが正解」は一概に言えない
データの性質とビジネス要件に合わせて判断することが重要

フィルタリングと集計

🔍 フィルタリング：条件で絞り込む
# アルコール度数が 13.5 以上
high_alc = df[df['alcohol'] >= 13.5]

# 複数条件（& で AND）
filtered = df[
    (df['alcohol'] >= 13.5) &
    (df['color_intensity'] >= 5.0)
]
実務例：「関東の」「女性の」「30代の」データだけを抽出
📊 groupby：グループごとに集計
# 品種ごとの平均を集計
df.groupby('品種')[[
    'alcohol', 'proline'
]].mean().round(2)

# ソート
df.sort_values('proline',
    ascending=False).head()
実務例：「都道府県ごと」「月ごと」の売上平均
