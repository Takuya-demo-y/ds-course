### スライド 02-03｜なぜグラフを作るのか（EDAとは）

**数値だけでは見えないものが、グラフにすると見える**

**EDA（探索的データ分析）とは**
> モデルを作る前に、データの「形・傾向・外れ値・変数間の関係」を  
> グラフで可視化して把握する作業

**数値で見たとき vs グラフで見たとき**

```
数値だけ（Ch.1の集計表）         グラフにすると
──────────────────────          ──────────────
Barolo:  alcohol 平均 13.7      「他の品種と明らかに分布が違う」
Grignolino: alcohol 平均 12.3   「2つの山がある？」
Barbera: alcohol 平均 12.3      「BarbaraとGrignolinoは似ている」
                                → 数値を見るだけではわからなかった
```

**EDAをしないと何が起きるか**

| EDAなし | EDAあり |
|--------|--------|
| 意味のない変数でもモデルに突っ込む | 効きそうな変数を絞り込んでからモデルを作れる |
| 「なぜこの精度か」を説明できない | グラフから仮説を立て、モデルで検証できる |
| 後から「データが変だった」と気づく | 外れ値・欠損を事前に発見できる |

**スライドノート（講師向け）**
> 「EDAはモデルを作る前の"下調べ"です。現場では"EDAをちゃんとやった人"と"やっていない人"で、モデルの説明力に大きな差が出ます」と話す。

### スライド 02-04｜【概念①】グラフは「何を見たいか」で選ぶ

**グラフの種類は目的で決まる。コードは後から調べればいい**

**4種類のグラフと使い分け**

```
見たいこと                使うグラフ         一言説明
──────────────────────────────────────────────────
1変数の分布（広がり方）  → ヒストグラム     「この値はどこに集まっているか」
グループ間の比較         → 箱ひげ図         「品種によってどれくらい違うか」
2変数の関係             → 散布図           「一方が増えたら他方も増えるか」
全変数の相関を一覧        → ヒートマップ     「どの変数同士が関係しているか」
```

**グラフ選びの判断フロー**

```
「何を見たいか？」
     ↓
1変数だけ？ → ヒストグラム
     ↓
2変数の関係？ → 散布図
     ↓
グループを比べたい？ → 箱ひげ図
     ↓
全部まとめて見たい？ → ヒートマップ
```

**重要な考え方**
> グラフのコードはAIに生成させてOK。「何を見たいか」を決めるのが自分の仕事。

**スライドノート（講師向け）**

### スライド 02-05｜【概念②】ヒストグラムから「分布の形」を読む

**グラフを「見る」のではなく「読む」**

**ヒストグラムの読み方**

```
              ヒストグラムの形と意味

左に偏っている   |▊▊▊▊▊▊▊▊         低い値が多い
                |

右に偏っている          ▊▊▊▊▊▊▊▊|  高い値が多い
                                  |

2つの山がある   |▊▊▊    ▊▊▊▊|     2つのグループが混在している可能性
                |            |
```

**品種別に重ねると何が見えるか**

```
品種を色分けしてヒストグラムを重ねると…

→ 品種ごとに「山の位置」が違う = 品種によって特徴量が違う
→ 山が重なっている = その変数では品種を区別しにくい
→ 山が離れている = その変数で品種を区別できそう（= モデルに効く変数の候補）
```

### スライド 02-06｜【概念③】相関係数は「関係の強さ」を数値化したもの

**2つの変数がどれくらい「一緒に動くか」を -1〜+1 の数値で表す**

**相関係数の読み方**

```
r ≈ +1    右肩上がり   一方が増えると他方も増える
                      例：身長と体重

r ≈  0    バラバラ     2変数に関係がない
                      例：靴のサイズと成績

r ≈ -1    右肩下がり   一方が増えると他方は減る
                      例：気温と暖房費
```

**ヒートマップで全変数を一覧する**

```
ヒートマップの読み方：

赤い（r が高い）   → 正の相関が強い → 予測に効く変数の候補
青い（r が低い）   → 負の相関が強い → 逆方向に動く
白い（r ≈ 0）     → 関係なし
```

**絶対に覚えてほしいこと：相関 ≠ 因果関係**

```
有名な例：アイスの売上 と 溺死者数 は正の相関がある
                  ↑
  なぜ？  → どちらも「夏（気温）」が原因。アイスが溺死を引き起こすわけではない
```

> 相関があっても「原因と結果の関係」とは限らない。解釈に注意が必要。

**発問②（仮説発問）**
> 💬 「ヒートマップを見て、Ch.4のモデルで最も効きそうな変数はどれですか？理由も教えてください」

**スライドノート（講師向け）**
> 「相関≠因果」は統計の基本。受講生が「相関が高いから影響している」と思い込まないよう、アイスと溺死の例で印象づける。発問②は演習問3への布石なので、答えをノートにメモさせる。

### スライド 02-07｜【概念④】EDAのゴールは「仮説の言語化」

**グラフを作るのは手段。ゴールは「グラフから言葉を引き出すこと」**

**仮説の書き方フォーマット**

```
「〇〇という特徴量は、△△という理由から、品種の分類に効くと思う」

例：
  「プロリン含有量は、Baroloだけ値が突出して高く、
   他の品種と数値の範囲がほとんど重ならないため、
   品種を分類するモデルで最も重要な変数になると思う」
```

**良い仮説と弱い仮説の違い**

| 弱い仮説 | 良い仮説 |
|---------|---------|
| 「相関が高いから使えそう」 | 「〇〇変数はX品種と他品種で値の範囲が重ならないから分類に効きそう」 |
| 「グラフがきれいに分かれていた」 | 「散布図でX品種だけ右上の領域に集まっており、2変数の組み合わせで分離できそう」 |

**なぜ仮説を立てるのか**

> Ch.4でモデルを作ったとき、「特徴量重要度」と仮説を照合できる。  
> 「仮説が当たっていた/外れた」という発見が、データへの理解を深める。

**スライドノート（講師向け）**
> 「仮説発問②で出てきた意見を"こういう形で書けばいい"と具体化してあげる。演習問3でこのフォーマットを使って書かせる。正解なし、思ったことを書けばいい」と伝える。

### スライド 02-10｜Ch.2 まとめ

**今日理解した「考え方」と「コード」の対応**

**概念の流れ（これが核心）**

```
① 目的でグラフを選ぶ    → 「何を見たいか」でヒスト/箱ひげ/散布図/ヒートマップ

② グラフを「読む」      → 形・偏り・2つの山・外れ値に注目

```

| 散布図 | `plt.scatter(x, y)` | 2変数の関係 |
| ヒートマップ | `sns.heatmap(df.corr())` | 全変数の相関 |
