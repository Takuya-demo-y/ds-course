## S7｜【座学 1/2】エラーは「道しるべ」

> 💬 **発問：** マクロが動かなかったとき、これまでどう対処していましたか？
> （コードを全部消した / 諦めた / 誰かに聞いた … ）

> 🍳 **たとえ話：レシピ通りに作ったのに、料理がおいしくない**
>
> 材料も揃っている。手順も間違っていない。でも何かがおかしい。
> このとき必要なのは「全部捨てて作り直す」ではなく、
> **「どこが間違っているかを冷静に確認する作業」** ── それがデバッグです。
>
> **エラーは「壊れた」サインではなく、「ここを直して」というヒントです。**

### エラーの3種類

| 🔴 コンパイルエラー | 🟠 実行時エラー | 🟡 論理エラー |
|---|---|---|
| 実行 **前** に発覚 | 実行 **中** に発生 | エラーは **出ない** |
| 文法の間違い | 番号付きのエラーが出る | でも結果が間違い |
| 例：括弧の閉じ忘れ | 例：ファイルが見つからない | 例：列番号が1つずれていた |

> **最も厄介なのは論理エラー。** 今日の演習で実際に体験します。

## S8｜【座学 2/2】デバッグの3つの武器

| 📋 武器① 記録する | ⏸ 武器② 一時停止 | 👣 武器③ 1行ずつ実行 |
|---|---|---|
| 処理の途中で変数の値を「ログ」として記録する | 任意の行で処理を一時停止させる | コードを1行ずつ手動で進める |
| 処理が止まらないのでループ内の確認に最適 | 停止中に変数の値をマウスオーバーで確認 | どの行で値が変わるかを目で追いながら特定できる |

> **大事な習慣：** コードを書いたら、いきなり実行せず、まず1行ずつ動かして確認する。

## S9｜【ケース演習 1/2】武器① Debug.Print の使い方

```vba
Sub 売上計算()
    Dim i As Long
    For i = 1 To 10

        ' ↓ この1行を追加するだけ。Ctrl+G でイミディエイトウィンドウに出力される
        Debug.Print "行:" & i & "  商品名:" & Cells(i, 1).Value

        Cells(i, 4).Value = Cells(i, 2).Value * Cells(i, 3).Value
    Next i
End Sub
```

> 💡 Debug.Print は処理を止めずに値を記録できる。MsgBox と違ってクリックが不要なのでループ内での確認に最適。

| 開き方 | 確認が終わったら | 何を出力するか |
|---|---|---|
| `Ctrl + G` → イミディエイトウィンドウ | `'` でコメントアウト（消さなくてよい） | ループカウンター変数 i、セルの値・計算途中の値 |

## S13｜【座学 1/2】「同じ処理を繰り返す」問題

> 💬 **発問：** 同じ計算や処理を、コードの中に何度もコピペして書いたことはありますか？
> そのとき「なんかスッキリしないな…」と感じませんでしたか？

> 🏪 **たとえ話：コンビニのレジマニュアル**
>
> 新人が入るたびに「口頭でゼロから説明する」と、説明者によって内容がバラバラ。

>

| ❌ 今の状態 | ✅ Function を使うと |
|---|---|
| 同じ処理が3か所にコピペされている | 処理が1か所にまとまる |
| 1か所直しても残り2か所は古いまま | 直す場所が1か所だけになる |
| コードが長くて読みにくい | コードが短く、意図が伝わりやすい |

## S14｜【座学 2/2】Sub と Function の違い

| | 📋 Sub ─「仕事をするだけ」 | 🔄 Function ─「答えを返す」 |
|---|---|---|
| 役割 | 呼ばれたら処理を実行する | 呼ばれたら処理して **結果を返す** |
| 戻り値 | **返さない** | **あり** |
| 用途 | ボタンで動かすメインの処理 | 「何かを調べて答えを教えてほしい」場面 |
| 例 | データを転記して、完了メッセージを出す | 「この列の最終行は何行目？」「この売上金額は何評価？」 |

> **判断基準：**「答えを返す必要があるか？」→ あれば Function、なければ Sub

## S15｜【ケース演習 1/3】Function の書き方

```vba
' ─── Function の定義 ───────────────────────────────────────
Function 最終行取得(ws As Worksheet, col As Long) As Long
'        ↑ 関数名     ↑ 引数（渡す値）                ↑ 返す値の型

    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
'   ↑ 関数名 = 戻り値  ← この書き方で「答え」を返す

End Function

' ─── 呼び出し側（Sub の中から使う）──────────────────────────
Sub 集計()
    Dim lastRow As Long
    lastRow = 最終行取得(ActiveSheet, 1)  ' A列の最終行番号が返ってくる
    Debug.Print lastRow                   ' 確認
End Sub
```

> 💡「関数名 = 戻り値」という書き方が最初は不思議に感じる。「関数名という変数に結果を代入する」イメージで覚える。

> ⚠️ 戻り値の代入を忘れると Function が 0 や空文字を返してしまう。`End Function` の前に必ず書く。

## S16｜【ケース演習 2/3】Before / After で比べる

**❌ Before（重複コード）**

```vba
Sub 集計()
    ' 同じコードが3回登場
    row1 = Cells(Rows.Count, 1).End(xlUp).Row
    row2 = Cells(Rows.Count, 2).End(xlUp).Row
    row3 = Cells(Rows.Count, 3).End(xlUp).Row
End Sub
```

**✅ After（Function に切り出す）**

```vba
Function 最終行取得(ws As Worksheet, col As Long) As Long
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
End Function

Sub 集計()
    row1 = 最終行取得(ws, 1)
    row2 = 最終行取得(ws, 2)
    row3 = 最終行取得(ws, 3)
End Sub
```

## S23｜【ケース演習 1/3】シートの参照・作成・削除

```vba
' ✅ シートを変数に入れて操作する（Set が必要）
Dim ws As Worksheet
Set ws = ThisWorkbook.Worksheets("集計")
ws.Cells(1, 1).Value = "タイトル"

' ✅ シートの存在確認と削除→再作成
Dim sheetName As String
sheetName = "集計"

Application.DisplayAlerts = False   ' 削除確認ダイアログを非表示
On Error Resume Next
Set ws = Worksheets(sheetName)
On Error GoTo 0                     ' ← 必ずここでリセット！

If Not ws Is Nothing Then
    ws.Delete
    Set ws = Nothing
End If

Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = sheetName
Application.DisplayAlerts = True    ' ← 忘れずに元に戻す
```

> 💡 `On Error Resume Next` は「この1行だけエラーを無視」という限定的な使い方。直後に `On Error GoTo 0` でリセットを忘れずに。

## S24｜【ケース演習 2/3】ファイルを「開く前に確認する」パターン

```vba
Sub ファイルを安全に開く()
    Dim folderPath As String
    Dim fileName   As String
    Dim filePath   As String

    folderPath = ThisWorkbook.Path & "\支店データ\"
    fileName   = Dir(folderPath & "*.xlsx")   ' 1日目で学んだ Dir

    Do While fileName <> ""

        filePath = folderPath & fileName

        ' ★ 開く前に存在確認する（Dir で確認）
        If Dir(filePath) <> "" Then
            Workbooks.Open filePath
            ' 処理
            ActiveWorkbook.Close SaveChanges:=False
        Else
            MsgBox fileName & " が見つかりません。スキップします。"
        End If

        fileName = Dir()   ' 次のファイルへ
    Loop
End Sub
```

> 💡 `Dir(filePath) <> ""` でファイルの存在を確認。**1日目に学んだ If と Dir だけで書ける** ので新しい文法は不要。

## S29｜【コード解説】処理を速くする・安全にする定番コード

```vba
Sub 支店別売上集計()

    ' ─── 処理前：画面更新・自動計算を一時停止（速度向上）───
    Application.ScreenUpdating = False   ' 画面をフリーズさせて高速化
    Application.DisplayAlerts  = False   ' 確認ダイアログを非表示

    ' ─── 実際の処理 ───
    ' ...

    ' ─── 処理後：必ず元に戻す ───
    Application.ScreenUpdating = True    ' ← これを忘れると画面が固まったまま！
    Application.DisplayAlerts  = True

    MsgBox "完了しました！", vbInformation

End Sub
```

> 💡 `ScreenUpdating = False` にすると画面更新がなくなり処理が劇的に速くなる。100ファイル処理なら体感速度が大きく変わる。

> ⚠️ エラーが起きても True に戻す処理を入れること。忘れると Excel の画面が更新されなくなる。
