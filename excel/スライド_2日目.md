## 座学 1/2｜エラーは「道しるべ」

### エラーは「壊れた」サインではなく「ここを直して」というヒント

**ちょっと考えてみよう：** マクロが動かなかったとき、これまでどう対処していましたか？  
（コードを全部消した ／ 諦めた ／ 誰かに頼った…）

**たとえ話：レシピ通りに作ったのに、料理がおいしくない**

> 材料も揃っている。手順も間違っていない。でも何かがおかしい。  
> このとき必要なのは「全部捨てて作り直す」ではなく、  
> 「どこが間違っているかを冷静に確認する作業」── それがデバッグです。

---

## 座学 2/2｜エラーの3種類

### まずエラーの種類を知ることが第一歩

| | コンパイルエラー | 実行時エラー | 論理エラー |
|---|---|---|---|
| **発生タイミング** | 実行「前」に発覚 | 実行「中」に発生 | エラーは「出ない」 |
| **症状** | 文法の間違い・VBEが赤く示す | 番号付きダイアログが表示される | 結果が間違っている |
| **例** | End If の閉じ忘れ・括弧ミス | ファイルが見つからない | 列番号が1つずれ・ループが1行多い |
| **対処** | VBEが赤く示す → すぐ気づける | エラー番号を検索して調べる | Debug.Print と F8 で追う |

> 最も厄介なのは **論理エラー** です。今日の演習で実際に体験します。

---

## 座学 補足｜デバッグの3つの武器

### 問題が起きたときに使う3つの道具

| 武器 | 操作 | 用途 |
|---|---|---|
| **Debug.Print** | `Ctrl + G` でイミディエイトウィンドウを開く | 変数の値を記録。処理を止めないのでループ内の確認に最適 |
| **ブレークポイント** | `F9` でセット → `F5` で実行 → 停止 | 任意の行で処理を一時停止。停止中に変数をマウスオーバーで値を確認 |
| **ステップ実行** | `F8` を押すたびに1行ずつ実行 | 黄色ハイライトが次の行に移動。どの行で値が変わるかを目で追える |

> **習慣：** コードを書いたら、いきなり実行せず、まず1行ずつ動かして確認する。

## ケース演習 1/3｜Debug.Print の使い方

### ループ内の値確認には MsgBox より Debug.Print が便利

```vba
Sub 売上計算()
    Dim i As Long
    For i = 1 To 10

        ' 下の行のコメント（'）を外すと、値がイミディエイトウィンドウに出力される
        ' Debug.Print "行:" & i & "  商品名:" & Cells(i, 1).Value

        Cells(i, 4).Value = Cells(i, 2).Value * Cells(i, 3).Value
    Next i
End Sub
```

| 操作 | 内容 |
|---|---|
| イミディエイトウィンドウを開く | `Ctrl + G` |
| 確認が終わったら | `'` でコメントアウト（消さなくてよい） |
| 何を出力するか | ループ変数 i、セルの値、計算途中の値 |

> MsgBox と違ってクリックが不要。ループ内の確認には Debug.Print が断然便利です。

## ケース演習 2/3｜ブレークポイント（F9）

### 怪しい行の直前で止めて、変数の中身を確認する

**使い方**

1. 止めたい行の**左端をクリック** → 赤丸 ● が表示される  
2. `F5` で実行 → その行でピタッと停止する  
3. 変数にマウスをかざす → 今の値がポップアップで表示される  

```vba
● For i = 2 To lastRow          ' ← ここで止まる
      数量 = Cells(i, 2).Value
      単価 = Cells(i, 3).Value  ' ← i にカーソルを当てると値が出る
      ...
```

## 座学 2/2｜Sub と Function の違い

### 「答えを返す必要があるか？」で使い分ける

| | Sub | Function |
|---|---|---|
| 役割 | 仕事をするだけ | 答えを返す |
| 戻り値 | なし | あり |
| 主な用途 | データを転記して完了メッセージを出す | 「この列の最終行は何行目？」「この売上金額は何評価？」 |

> **判断基準：** 「答えを返す必要があるか？」  
> → あれば **Function**、なければ **Sub**

## ケース演習 1/3｜Function の書き方

### 「関数名 = 戻り値」という形で答えを返す

```vba
' Function の定義
Function 最終行取得(ws As Worksheet, col As Long) As Long
'        関数名     引数（渡す値）                  返す値の型

    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
'   関数名 = 戻り値  ← この書き方で「答え」を返す

End Function

' 呼び出し側（Sub の中から使う）
Sub 集計()
    Dim lastRow As Long
    lastRow = 最終行取得(ActiveSheet, 1)  ' A列の最終行番号が返ってくる
    Debug.Print lastRow                   ' Ctrl+G で確認
End Sub
```

> 「関数名 = 戻り値」という書き方が最初は不思議に感じます。  
> 「関数名という変数に結果を代入する」イメージで覚えましょう。  
> End Function の前に戻り値を代入し忘れると、0 や空文字が返ってきます。

## ケース演習 2/3｜Before / After

### Function に切り出すと、修正箇所が1か所になる

**Before — 重複コード（修正が3か所必要）**

```vba
Sub 集計()
    row1 = Cells(Rows.Count, 1).End(xlUp).Row  ' 同じコードが
    row2 = Cells(Rows.Count, 2).End(xlUp).Row  ' 3回繰り返し
    row3 = Cells(Rows.Count, 3).End(xlUp).Row  ' 直すなら全部直す
End Sub
```

**After — Function に切り出す（修正は1か所）**

```vba
Function 最終行取得(ws As Worksheet, col As Long) As Long
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
End Function

Sub 集計()
    row1 = 最終行取得(ws, 1)  ' 呼び出しだけ
    row2 = 最終行取得(ws, 2)
    row3 = 最終行取得(ws, 3)
End Sub
```

## 座学 2/3｜シートを「変数」で管理する

### シート名が変わっても、変数名だけ直せばよい

**たとえ話：担当者の名前ではなく「役割」で呼ぶ**

> 「山田さんにお願い」と言うと、山田さんが変わったら全員に周知が必要。  
> 「集計担当にお願い」と言えば、誰が担当になっても指示の仕方は変わらない。

| 直接名前で指定（修正が全箇所必要） | 変数に入れてから操作（修正は1か所） |
|---|---|
| `Worksheets("集計").Cells(1,1)` | `Dim ws As Worksheet` |
| `Worksheets("集計").Cells(2,1)` | `Set ws = Worksheets("集計")` |
| `Worksheets("集計").Cells(3,1)` | `ws.Cells(1,1)` / `ws.Cells(2,1)` |

> **`Set` が必要** ── これだけ覚えましょう。

## 座学 3/3｜エラーを「防ぐ」考え方

### ファイルを開く前に「存在確認」する習慣をつける

**たとえ話：ドアを開ける前にノックする**


## ケース演習 1/3｜シートの参照・作成・削除

### シートの存在確認 → 削除 → 再作成 のパターンを覚える

```vba
' シートを変数に入れて操作する（Set が必要）
Dim ws As Worksheet
Set ws = ThisWorkbook.Worksheets("集計")
ws.Cells(1, 1).Value = "タイトル"

' シートの存在確認と削除 → 再作成
Application.DisplayAlerts = False    ' 削除確認ダイアログを非表示
On Error Resume Next
Set ws = Worksheets("集計")
On Error GoTo 0                      ' 必ずここでリセット！

If Not ws Is Nothing Then
    ws.Delete
    Set ws = Nothing
End If

Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = "集計"
Application.DisplayAlerts = True     ' 忘れずに元に戻す
```

> `On Error Resume Next` は「この1行だけエラーを無視」という限定的な使い方。  
> 直後に `On Error GoTo 0` でリセットを忘れずに。

## ケース演習 2/3｜ファイルを「開く前に確認する」

### Dir で存在確認してから開く — 実務でよく使うパターン

```vba
Sub ファイルを安全に開く()
    Dim folderPath As String
    Dim fileName   As String
    Dim filePath   As String

    folderPath = ThisWorkbook.Path & "\支店データ\"
    fileName   = Dir(folderPath & "*.xlsx")    ' 1日目で学んだ Dir

    Do While fileName <> ""

        filePath = folderPath & fileName

        ' 開く前に存在確認する（Dir で確認）
        If Dir(filePath) <> "" Then
            Workbooks.Open filePath
            ' 処理
            ActiveWorkbook.Close SaveChanges:=False
        Else
            MsgBox fileName & " が見つかりません。スキップします。"
        End If

        fileName = Dir()    ' 次のファイルへ
    Loop
End Sub
```

## 総合演習｜処理を速くする・安全にする

### ScreenUpdating をオフにすると処理が速くなる

```vba
Sub 支店別売上集計()

    ' 処理前：画面更新・自動計算を一時停止（速度向上）
    Application.ScreenUpdating = False   ' 画面をフリーズして高速化
    Application.DisplayAlerts  = False   ' 確認ダイアログを非表示

    ' 実際の処理 ...

    ' 処理後：必ず元に戻す
    Application.ScreenUpdating = True    ' これを忘れると画面が固まったまま
    Application.DisplayAlerts  = True

    MsgBox "完了しました！", vbInformation

End Sub
```

> `ScreenUpdating = False` にすると画面更新がなくなり処理が速くなります。  
> ファイル数が多いほど体感速度の差が大きくなります。
