## S11 ｜ ケース演習 1/3 ｜ Debug.Print の使い方

```vba
Sub 売上計算()
    Dim i As Long
    For i = 1 To 10

        ' 下の行のコメント（'）を外すと、値がイミディエイトウィンドウに出力される
        ' Debug.Print "行:" & i & "  商品名:" & Cells(i, 1).Value

        Cells(i, 4).Value = Cells(i, 2).Value * Cells(i, 3).Value
    Next i
End Sub
```

| 操作 | 内容 |
|---|---|
| イミディエイトウィンドウを開く | `Ctrl + G` |
| 確認が終わったら | `'` でコメントアウト（消さなくてよい） |
| 何を出力するか | ループ変数 i、セルの値、計算途中の値 |

> MsgBox と違ってクリックが不要。ループ内の確認には Debug.Print が断然便利です。

## S19 ｜ ケース演習 1/3 ｜ Function の書き方

```vba
' Function の定義
Function 最終行取得(ws As Worksheet, col As Long) As Long
'        関数名     引数（渡す値）                  返す値の型

    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
'   関数名 = 戻り値  ← この書き方で「答え」を返す

End Function

' 呼び出し側（Sub の中から使う）
Sub 集計()
    Dim lastRow As Long
    lastRow = 最終行取得(ActiveSheet, 1)  ' A列の最終行番号が返ってくる
    Debug.Print lastRow                   ' Ctrl+G で確認
End Sub
```

> 「関数名 = 戻り値」という書き方が最初は不思議に感じます。  
> 「関数名という変数に結果を代入する」イメージで覚えましょう。

> End Function の前に戻り値を代入し忘れると、0 や空文字が返ってきます。

## S20 ｜ ケース演習 2/3 ｜ Before / After で比べる

```
  Before（重複コード）                After（Function に切り出す）

  Sub 集計()                          Function 最終行取得(
      '── 同じコードが3回 ──               ws As Worksheet,
      row1 = Cells(Rows.Count,1)          col As Long) As Long
                .End(xlUp).Row              最終行取得 =
      row2 = Cells(Rows.Count,2)              ws.Cells(
                .End(xlUp).Row                  ws.Rows.Count,
      row3 = Cells(Rows.Count,3)                col).End(xlUp).Row
                .End(xlUp).Row          End Function
  End Sub
                                        Sub 集計()
  問題点:                                   row1 = 最終行取得(ws, 1)
  - 修正が必要なら 3 か所全部直す           row2 = 最終行取得(ws, 2)
  - 1 か所直し忘れるとバグになる            row3 = 最終行取得(ws, 3)
  - コードが長くて読みにくい            End Sub
```

## S27 ｜ ケース演習 1/3 ｜ シートの参照・作成・削除

```vba
' シートを変数に入れて操作する（Set が必要）
Dim ws As Worksheet
Set ws = ThisWorkbook.Worksheets("集計")
ws.Cells(1, 1).Value = "タイトル"

' シートの存在確認と削除 → 再作成
Application.DisplayAlerts = False    ' 削除確認ダイアログを非表示
On Error Resume Next
Set ws = Worksheets("集計")
On Error GoTo 0                      ' 必ずここでリセット！

If Not ws Is Nothing Then
    ws.Delete
    Set ws = Nothing
End If

Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = "集計"
Application.DisplayAlerts = True     ' 忘れずに元に戻す
```

> `On Error Resume Next` は「この 1 行だけエラーを無視」という限定的な使い方。  
> 直後に `On Error GoTo 0` でリセットを忘れずに。

## S28 ｜ ケース演習 2/3 ｜ ファイルを「開く前に確認する」パターン

```vba
Sub ファイルを安全に開く()
    Dim folderPath As String
    Dim fileName   As String
    Dim filePath   As String

    folderPath = ThisWorkbook.Path & "\支店データ\"
    fileName   = Dir(folderPath & "*.xlsx")    ' 1 日目で学んだ Dir

    Do While fileName <> ""

        filePath = folderPath & fileName

        ' 開く前に存在確認する（Dir で確認）← Block 3 で学んだパターン
        If Dir(filePath) <> "" Then
            Workbooks.Open filePath
            ' 処理
            ActiveWorkbook.Close SaveChanges:=False
        Else
            MsgBox fileName & " が見つかりません。スキップします。"
        End If

        fileName = Dir()    ' 次のファイルへ
    Loop
End Sub
```

```vba
Sub 支店別売上集計()

    ' 処理前：画面更新・自動計算を一時停止（速度向上）
    Application.ScreenUpdating = False   ' 画面をフリーズして高速化
    Application.DisplayAlerts  = False   ' 確認ダイアログを非表示

End Sub
```
