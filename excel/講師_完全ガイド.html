演習1 答え（クリックで展開）

バグ1： For i = 1 To lastRow → For i = 2 To lastRow
ループが1行目（ヘッダー行）から始まっているため、ヘッダーを数値として処理しようとしてしまう。

バグ2： Cells(i, 5).Value = 売上 → Cells(i, 4).Value = 売上
書き込み先の列番号が E列（5）になっているが、正しくは D列（4）。

For i = 2 To lastRow     ' ← 2から始める
    数量 = Cells(i, 2).Value
    単価 = Cells(i, 3).Value
    売上 = 数量 * 単価
    Cells(i, 4).Value = 売上  ' ← 列4（D列）に書き込む
Next i

🎤 「Function を使う前と使った後で、コードの読みやすさがどう変わるか見てください。最終行取得(ws, 1) と書くだけで意図が伝わります」

巡回時の声掛け

  • 「売上評価 = "A" のように、関数名に代入する形で戻り値を返してください」

演習3 答え（クリックで展開） 
Function 売上評価(売上金額 As Long) As String
    If     売上金額 >= 1000000 Then
        売上評価 = "A"
    ElseIf 売上金額 >= 500000 Then
        売上評価 = "B"
    Else
        売上評価 = "C"
    End If
End Function

Function 最終行取得(ws As Worksheet, col As Long) As Long
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
End Function

Sub 評価を入力する()
    Dim ws      As Worksheet
    Dim i       As Long
    Dim lastRow As Long

    Set ws = ThisWorkbook.Worksheets("売上データ")
    lastRow = 最終行取得(ws, 4)   ' D列（売上金額）を基準

    For i = 2 To lastRow
        ws.Cells(i, 5).Value = 売上評価(ws.Cells(i, 4).Value)
    Next i

    MsgBox "評価の入力が完了しました！"
End Sub

❓よくある質問と回答

Call は省略できますか？
Call プロシージャ名 は プロシージャ名 だけでも動きます。ただし Call をつけると「Sub の呼び出し」だと明示できるので読みやすくなります。
Function と Sub はどちらを使えばよいですか？
「結果を返す必要があるか」で判断します。計算・判定など「何かを返す処理」は Function、実行するだけの処理は Sub を使います。
ThisWorkbook と ActiveWorkbook の違いは？
ThisWorkbook はそのマクロが書かれているブック（固定）。ActiveWorkbook は現在アクティブなブック（変動する）。外部ファイルを開く処理では ThisWorkbook を使わないと意図しないブックを操作してしまうことがある。
On Error GoTo 0 と On Error GoTo ErrorHandler の違いは？
On Error GoTo 0 は「エラーハンドリングを解除してデフォルト状態に戻す」コードです。On Error Resume Next や個別 On Error GoTo を使った後の範囲を限定するために使います。
ByVal と ByRef はどちらを使えばよいですか？
迷ったら ByVal を使います。ByRef は「意図して元の変数を変えたいとき」だけ使います。VBA のデフォルトは ByRef なので、明示的に書く習慣が大切です。
マクロが非常に遅いのですが、なぜですか？
主な原因は「セルへのアクセス回数が多い」こと。Application.ScreenUpdating = False で改善することも多いですが、根本的な解決は「配列にデータを一括格納してから処理する」方法です（配列は中級以降で学ぶテーマです）。
