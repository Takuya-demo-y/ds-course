🎯本日の伝えたいメッセージ

S6 セクションヘッダー

「最初のブロックはデバッグです。コードが思い通りに動かないとき、どこを見ればいいかを学びます。実は今日の演習にはわざとバグを仕込んだコードが入っています。それを3つの武器で見つけてもらいます。」

S7 【座学 1/2】エラーの3種類 10分

座学スライドの鉄則：コードの解説はしない。
「なぜ必要か」「どんな状況で起きるか」のイメージを作ることに集中してください。
「エラーには3種類あります。コンパイルエラーは文法ミスなのでVBEがすぐに教えてくれます。一番怖いのは論理エラーです。エラーは出ないのに結果が間違っている。今日の演習でこの論理エラーを体験してもらいます。」
「経験ある方に聞いてみてください：『コードは動いたのに、出てきた数字がおかしかった経験はありますか？』と。論理エラーの経験談を引き出すと理解が深まります。

S8 【座学 2/2】デバッグの3武器 10分

「レシピ通りに料理したのになぜかまずい。このとき全部捨てて作り直しますか？しないですよね。どこが悪いか確認するはずです。デバッグも同じです。3つの武器を使って『どこがおかしいか』を確認します。Debug.Print が記録する武器、ブレークポイントが一時停止する武器、F8が1行ずつ実行する武器です。」

S9 【ケース 1/2】Debug.Print の使い方 10分

「Debug.Print はコードの中に1行追加するだけです。Ctrl+G でイミディエイトウィンドウを開いて確認します。MsgBoxと違って処理が止まらないので、ループの中で変数の変化を追うのに最適です。確認が終わったらコメントアウトするだけで、削除しなくていいです。」

S10 【ケース 2/2】ブレークポイント・F8 10分

「ブレークポイントはコードを一時停止させる機能です。止まった状態で変数の値をマウスオーバーするだけで確認できます。そこからF8を押すと1行ずつ進められます。この『止めてから1行ずつ』が基本のデバッグパターンです。では演習に入りましょう。」
演習1 前の確認事項：
① 全員の `01_デバッグ演習/デバッグ演習.xlsx` が開けているか確認
② `VBAコード_バグあり.txt` の内容を VBE に貼り付けるよう指示
③ 「今から20分で2箇所のバグを見つけてください」と明確に伝える
④ 解答は15分後に全体でシェアする、と事前予告しておく

演習1 の答え（講師向け）

バグ番号場所間違い正解
バグ1 For ループの開始値 For i = 2 To lastRow → i = 0 など開始が間違い For i = 2 To lastRow（2行目からデータ）
バグ2 売上金額の列参照 Cells(i, 3)（C列を参照） Cells(i, 2)（B列が正しい売上列）

S12 セクションヘッダー

「Block 1 お疲れさまでした。次は Function です。同じ処理を何度も書かない仕組みを作ります。これを覚えると、マクロを修正するときに1か所直すだけで全部に反映されるようになります。」

S13 【座学 1/2】同じ処理を繰り返す問題 10分

「コンビニのレジマニュアルの話をします。新人が入るたびに口頭でゼロから説明するのと、マニュアルを作っておくのでは全然違います。マニュアルがあれば修正は1か所で済む。コードも同じで、同じ処理をFunction にまとめておけば、修正は1か所だけです。」

S14 【座学 2/2】Sub と Function の違い 10分

「Sub は『仕事をするだけ』、Function は『答えを返す』という違いです。判断基準は『答えを返す必要があるか？』です。例えば『この列の最終行は何行目？』という質問に答えてもらいたいときはFunction を使います。」
よくある質問：「いつも Sub で書いてきたので Function が必要な場面がわかりません」
→ 「同じ処理を書いた瞬間にFunction候補」と伝える。コピペした瞬間に Function 化を検討するクセをつける。

S15 【ケース 1/3】Function の書き方 10分

「構文はシンプルです。一番のポイントは『関数名 = 戻り値』という書き方で答えを返すことです。Return文ではないので最初は違和感があるかもしれませんが、これがVBAの流儀です。End Function の前に必ず書いてください。」

S16 【ケース 2/3】Before / After 5分

「Before を見てください。同じコードが3回書かれています。もしこの処理に間違いがあったら3か所直す必要があります。After はFunction に切り出したので1か所だけです。コードの行数も短くなりますし、意図が読み取りやすくなります。」

S17 【ケース 3/3】演習2 30分

演習2 前の確認事項：

② 「まず STEP 1 だけ作って Debug.Print でテストしてから STEP 2 に進む」と明確に伝える
③ 全員同時に STEP 3 まで終わらなくてOK。STEP 2 まで終われば合格

演習2 のサポートポイント

❓よくある質問 Q&A

デバッグ関連

Debug.Print を書いたのに何も表示されません
Ctrl + G でイミディエイトウィンドウを開きましたか？VBE の表示メニューからでも開けます。
ブレークポイントを設定したのに止まりません
実行は F5 を使っていますか？ボタンから実行している場合はブレークポイントが機能します。VBE の「実行」メニューから実行してもOKです。
F8 でステップ実行中に変数の値が表示されません
マウスを変数名の上にゆっくりホバーさせてみてください。ポップアップが出るまで1〜2秒かかることがあります。または「ローカルウィンドウ」（表示メニューから）で一覧確認できます。

Function 関連

Function が常に 0 を返します
関数名 = 戻り値 の代入が書かれていますか？Return ではなく関数名に代入するのが VBA の流儀です。End Function の前に必ず書いてください。
Function を呼び出すとき引数は省略できますか？
省略したい引数は定義時に Optional 引数名 As 型 = デフォルト値 とします。全部の引数は省略できません。
Sub と Function は同じファイルに書いていいですか？
はい、同じ標準モジュールに書いてOKです。ただし大きくなってきたら modUtil という共通モジュールに Function をまとめると管理しやすくなります。

シート操作関連

Set を忘れたらどうなりますか？
「オブジェクト変数または With ブロック変数が設定されていません」というエラー（エラー番号91）が出ます。Dim ws As Worksheet の後は必ず Set ws = ... と書きましょう。
DisplayAlerts を False にしたまま処理が止まってしまいました
VBE のイミディエイトウィンドウで Application.DisplayAlerts = True と入力して Enter を押せば元に戻ります。
On Error Resume Next の後に On Error GoTo 0 を書く理由は？
On Error Resume Next はすべてのエラーを無視し続けます。リセットしないと、後続の処理で起きたエラーも全部無視されてしまいます。「1行だけ無視する」という使い方のために、直後に必ずリセットしてください。
