## 【Block 1】デバッグ入門

### エラーの3種類

| 種類 | タイミング | 具体例 |
|---|---|---|
| **コンパイルエラー** | 実行前（文法ミス） | `End If` が抜けている |
| **実行時エラー** | 実行中 | 存在しないファイルを開こうとした |
| **論理エラー** | エラーは出ないが結果が間違い | 列番号が1つずれていた |

> **ポイント：** エラーメッセージは「敵」ではなく「ヒント」です。  
> エラー番号はそのまま Google で検索できます。  
> 最も注意が必要なのは **論理エラー**（エラーが出ないまま結果が間違う）です。

---

### デバッグの3つの武器

#### 武器① `Debug.Print` ─ 処理を止めずに値を記録する

```vba
Sub 売上計算()
    Dim i As Long
    For i = 1 To 10
        Debug.Print "行:" & i & "  商品名:" & Cells(i, 1).Value  ' ← 追加するだけ
        Cells(i, 4).Value = Cells(i, 2).Value * Cells(i, 3).Value
    Next i
End Sub
```

- **開き方：** `Ctrl + G` でイミディエイトウィンドウを表示
- **特長：** `MsgBox` と違って処理が止まらない。ループ内での確認に最適
- **終わったら：** `'` でコメントアウトしておく（削除しなくてよい）

#### 武器② ブレークポイント（`F9`）─ 任意の行で一時停止

1. 止めたい行の左端をクリック → 赤丸 ● が表示される
2. `F5` で実行 → その行で停止
3. 停止中に変数にマウスオーバー → 現在値が表示される

#### 武器③ ステップ実行（`F8`）─ 1行ずつ実行

- `F8` を押すたびに1行ずつ実行。黄色ハイライトが次の行へ進む
- どの行で値が変わるかを**目で追いながら**問題を特定できる

> **組み合わせ技：** ブレークポイントで止めた後、`F8` で1行ずつ進めながら確認するのが基本パターンです。

## 【Block 2】Function（関数）とコードの再利用

### なぜ Function が必要か

同じ処理を複数箇所にコピペで書くと、修正が必要になったとき**全箇所を直す必要が生じます。**  
Function を使えば「1か所に書いて、何度でも呼び出す」構造にできます。

> **たとえ話：** 新人が入るたびに口頭でゼロから説明するより、  
> **「マニュアルを一度作っておけば」**、誰でも使えるし、修正は1か所だけで全員に反映される。

---

### Sub と Function の違い

| | `Sub` | `Function` |
|---|---|---|
| 戻り値 | なし | **あり** |
| 呼び出し方 | `Call プロシージャ名` | `変数 = 関数名(引数)` |
| 主な用途 | 一連の処理を実行する | 計算・判定結果を返す |

> **判断基準：「答えを返す必要があるか？」** → あれば Function、なければ Sub

---

### Function の書き方

```vba
' ─── 定義 ───
Function 最終行取得(ws As Worksheet, col As Long) As Long
    '                ↑ 引数                         ↑ 戻り値の型
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
    '  ↑ 関数名 = 戻り値  という形で「答え」を返す
End Function

' ─── 呼び出し ───
Sub 集計()
    Dim lastRow As Long
    lastRow = 最終行取得(ActiveSheet, 1)   ' A列の最終行番号が返ってくる
    Debug.Print lastRow
End Sub
```

> **注意：** `関数名 = 戻り値` の記述を忘れると Function が 0 や空文字を返してしまいます。  
> `End Function` の前に必ず書きましょう。

---

### Before / After で比べる

```vba
' ❌ Before（同じコードが3回コピペされている）
Sub 集計()
    row1 = Cells(Rows.Count, 1).End(xlUp).Row
    row2 = Cells(Rows.Count, 2).End(xlUp).Row
    row3 = Cells(Rows.Count, 3).End(xlUp).Row
End Sub

' ✅ After（Function に切り出すと、呼び出しが1行になる）
Sub 集計()
    row1 = 最終行取得(ActiveSheet, 1)
    row2 = 最終行取得(ActiveSheet, 2)
    row3 = 最終行取得(ActiveSheet, 3)
End Sub
```

## 【Block 3】シートの操作

### なぜシートを動的に作るのか

毎月「1月」「2月」…のシートを手動で作ると、名前のタイポ・作り忘れ・毎月同じ単純作業の繰り返しが起きます。  
マクロが自動で作れば、**何度でも正確に・ボタン1つで再実行できます。**

---

### シートを「変数」で管理する

```vba
' ✅ 変数に入れてから操作する（Set が必要）
Dim ws As Worksheet
Set ws = ThisWorkbook.Worksheets("集計")
ws.Cells(1, 1).Value = "タイトル"
```

> **重要：** オブジェクト変数には `Set` が必要です。忘れるとエラーになります。

---

### シートの作成・削除（定番パターン）

```vba
' ─── 既存シートを削除して再作成する定番パターン ───
Dim ws As Worksheet
Dim sheetName As String
sheetName = "集計"

Application.DisplayAlerts = False      ' 削除確認ダイアログを非表示

On Error Resume Next
Set ws = Worksheets(sheetName)
On Error GoTo 0                        ' ← 必ずここでリセット

If Not ws Is Nothing Then
    ws.Delete
    Set ws = Nothing
End If

Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = sheetName
Application.DisplayAlerts = True       ' ← 忘れずに元に戻す
```

### ファイルを開く前に確認する（1日目の知識で書ける）

```vba
Sub ファイルを安全に開く()
    Dim folderPath As String
    Dim fileName   As String

    folderPath = ThisWorkbook.Path & "\支店データ\"
    fileName   = Dir(folderPath & "*.xlsx")    ' 1日目で学んだ Dir

    Do While fileName <> ""
        ' ★ 開く前に存在確認（Dir で確認 ← If + Dir だけでOK）
        If Dir(folderPath & fileName) <> "" Then
            Workbooks.Open folderPath & fileName
            ' 処理
            ActiveWorkbook.Close SaveChanges:=False
        Else
            MsgBox fileName & " が見つかりません。スキップします。"
        End If
        fileName = Dir()
    Loop
End Sub
```

> **ポイント：** `On Error GoTo` のような新しい文法は不要。  
> **1日目に学んだ `If` と `Dir` だけで「安全に開く」が書けます。**

### パフォーマンス最適化の定番コード

```vba
Sub メインマクロ()

    ' ─── 処理前：画面・計算を一時停止（速度向上）───
    Application.ScreenUpdating = False   ' 画面更新を止める → 処理が劇的に速くなる
    Application.DisplayAlerts  = False   ' 確認ダイアログを非表示

    ' ─── 実際の処理 ───
    ' ...

    ' ─── 処理後：必ず元に戻す ───
    Application.ScreenUpdating = True    ' ← これを忘れると画面が固まったまま！
    Application.DisplayAlerts  = True

    MsgBox "完了しました！", vbInformation

End Sub
```

> **注意：** エラーが起きても必ず `True` に戻す処理を入れましょう。忘れると Excel の画面が更新されなくなります。
