ステップバイステップ

  1. Excelを開く　デバッグ演習.xlsx を開き、下のデモのように「売上データ」シートに商品名・数量・単価があることを確認します。
  2. エディタを開く　キーボードで Alt + F11 を押し、Visual Basic エディタ（VBE）を開きます。
  3. 標準モジュールを用意　メニューから「挿入」→「標準モジュール」を選び、空のモジュールを作ります。
  4. コードを貼る　VBAコード_バグあり.txt をメモ帳などで開き、すべてコピーして標準モジュールに貼り付けます。
  5. ウォームアップから試す　まず ウォームアップ_Debug練習 という Sub にカーソルを置き、F5 で実行します。Ctrl + G でイミディエイトウィンドウを開き、Debug.Print の出力を見てデバッグの感覚を掴みます。
  6. 本番のバグを探す　売上計算 を実行すると、意図と違う結果になります。コメントのヒントに沿って、ループの開始行と書き込む列番号の2か所を見直します。
  7. 動作確認　もう一度 売上計算 を実行し、D2:D11 を選択したときの合計が ¥2,670,000 か確認します。

7. 回答と解説（受講生の認知負荷を下げる読み方）

この章だけで「答え」と「意味」が追えるようにしました

上のセクション（演習1〜4）は問題の地図、ここから先は答えの地図です。迷子になりにくいよう、次の工夫を入れています。

  • 1演習＝最初に1文だけ（いま何を覚えればよいか）
  • 図 → 手順 → 答え表の順（いきなりコードを読ませない）
  • 「もう少しだけ」は折りたたみ（必要な人だけ開く）

おすすめの読み順（1回15分イメージ）

図で全体像→ 答え表を見る→ Excelで照合→ 折りたたみは後回し

コードの全文は各フォルダの VBAコード_解答.txt が正本です。このページは「どこを直すか／何を書くか」を視覚で先に掴むためのガイドです。

回答 演習1 — デバッグ（答え・図解）
いま1つだけ：データは2行目から回し、売上はD列（列番号4）に書く。

Excelの見方（列番号の地図）

A列1商品名
B列2数量
C列3単価
D列4売上（書く）
E列5（今回は使わない）

答え（修正は2か所だけ）

バグのあった行（イメージ）正解
For i = 1 To lastRow
1行目の「商品名」まで計算に入る		 For i = 2 To lastRow
データ開始行から
Cells(i, 5).Value = 売上
E列に書いてしまう		 Cells(i, 4).Value = 売上
D列に書く
正解の確認： 売上計算 実行後、D2:D11 を選択 → ステータスバーの合計が ¥2,670,000。
もう少しだけ：なぜ「2」なのか／なぜ「4」なのか

Cells(行, 列) の「列」は左から数えた番号です。見出し行は人間が読むための行で、計算の材料ではありません。だからループは2行目から。売上列は表の設計上D列なので、列番号は4です。

回答 演習2 — Function（答え・図解）
いま1つだけ：金額の判定は大きい条件から。最終行はD列基準、評価の出力はE列。

シート上の「読む列／書く列」

A〜C列1〜3今回はループで直接は使わない
D列4売上を読む
E列5評価を書く

STEPごとの答え（穴埋め番号つき）

STEP埋める内容（要点）
STEP1If 売上金額 >= 1000000 Then → "A" → ElseIf ... >= 500000 → "B" → Else → "C"
STEP2最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
STEP3lastRow = 最終行取得(ws, 4)　と　grade = 売上評価(sales)
動作確認の順番： まずイミディエイトで ? 売上評価(1500000) が A になること → そのあと 評価を入力する を実行 → E列に A/B/C。
もう少しだけ：なぜ「100万から先」に書くのか

もし「50万以上」を先に書くと、120万円も最初の条件に入って B になってしまいます。だから厳しい（大きい）条件から並べます。

回答 演習3 — シート操作（答え・図解）
いま1つだけ：日付から Month で「何月か」を取り、シート 「◯月」 の次の空行に A〜E をコピーする。

STEP1（月別シートを準備）の穴埋め答え

穴答え一言
①sheetName消すシート名は "1月"…"12月"
②Nothing「シートが取れたか」を Not ws Is Nothing で判定
③Nothing削除後は変数を空に戻す
④sheetName新シート名も同じ文字列

STEP2（振り分け）の穴埋め答え

穴答え一言
⑤sheetName転記先シートを名前で指定
⑥1（+ 1）A列の最終行の次の行＝空行
⑦〜⑩2, 3, 4, 5転記元の B〜E 列（行は i のまま）
まとめ実行： 月別振り分け実行 ＝ STEP1のあと STEP2。各月シートの件数の合計が「全データ」と一致すれば良い状態です。
もう少しだけ：「次の空行」はなぜ +1？

転記先シートにはヘッダーが1行目にあります。データが2行目まで入っているなら「最終行」は2。その次に書きたいので +1 で3行目、という考え方です。
