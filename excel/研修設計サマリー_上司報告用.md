# Excel VBA 研修 設計報告

**報告日：** 2026年4月23日
**対象研修：** Excel VBA 研修（2日間）
**報告者：**

---

## 1. 研修の目的

定型作業の繰り返しによる工数・ミスを削減し、  
社員が自力でExcelの自動化ツールを作れる状態をつくることを目的としています。

---

## 2. 研修概要

| 項目 | 内容 |
|---|---|
| 期間 | 2日間（Day 1 基礎・Day 2 実践） |
| 対象 | Excel を日常的に使用している社員（初学者〜初級者） |
| 形式 | 座学 + 演習 + 業務への落とし込みワーク |
| 最終成果物 | 自社業務に即したExcel自動化マクロのアクションプラン |

---

## 3. 研修の設計方針

研修を通じて伝えるメッセージは以下の1点に集約されます。

> **VBAは「コードを書く技術」ではなく、**  
> **「繰り返し作業から解放され、価値ある仕事に時間を使う」ための道具である。**

---

## 4. 2日間のカリキュラム構成

### Day 1（基礎）：「動くものを作る」

条件分岐・繰り返し処理・ファイル操作など、VBAの基本文法を習得。  
最終演習として複数ファイルを統合するマクロを作成。

### Day 2（実践）：「使えるものに磨く」

Day 1 で作ったものを、職場で実際に使えるレベルに引き上げる。

| ブロック | テーマ | 所要時間 |
|---|---|---|
| Opening | 全体像の共有・Day 1 振り返り | 20分 |
| Block 1 | デバッグ入門（エラーの読み方・修正方法） | 70分 |
| Block 2 | Function（処理の再利用・コードの整理） | 80分 |
| Block 3 | シートの操作（月次処理・ファイル自動収集） | 90分 |
| 総合演習 | 支店別売上集計ツールの完成（2日間の統合） | 120分 |
| Closing | 振り返り・自動化アクションプランの作成 | 20分 |

---

## 5. 各ブロックの学習ゴール

| ブロック | 研修後にできること |
|---|---|
| デバッグ | エラーの原因を自力で特定し、修正できる |
| Function | コードを再利用可能な形に整理できる |
| シート操作 | 月次シートの自動生成・データの自動振り分けができる |
| 総合演習 | 複数ファイルを横断する集計ツールを自分で仕上げられる |

---

## 6. 研修設計の特徴

**特徴 1：各ブロックを同じ3ステップで統一**

```
座学（概念理解）→ ケース演習（手を動かす）→ 業務への落とし込み（自分の業務に当てはめる）
```

知識の習得で終わらせず、「自分の業務に使えそう」という実感を毎回得られる設計にしています。

**特徴 2：業務への落とし込みを必須ステップとして設計**

各ブロック終了後に 5〜10 分の「業務落とし込みワーク」を実施します。  
研修後に職場で活用できるかどうかは、この時間の質に依存します。省略不可とします。

**特徴 3：総合演習で「自分のものにする」体験を作る**

最終演習では、実務に近いシナリオ（4支店 × 6か月のファイル自動集計）を扱います。  
2日間で学んだ知識をすべて組み合わせて完成させることで、自己効力感の向上を図ります。
