1日目の振り返り ─ このコードは何をしていますか？

Sub 集計()
    Dim i       As Long
    Dim lastRow As Long

    lastRow = Cells(Rows.Count, 1).End(xlUp).Row  ' ★ここ

    For i = 2 To lastRow
        Debug.Print Cells(i, 1).Value
    Next i
End Sub
★ の行は何をしていますか？（答えは次のスライド）

答え：最終行を取得している

lastRow = Cells(Rows.Count, 1).End(xlUp).Row
'              ↑ 列の一番下 ↑ 上に向かって  ↑ 行番号を返す
' → A列の最終データ行の行番号を取得する（定番パターン）
※ このパターンは今日の演習でも何度も使います

デバッグ入門

コードが動かないとき、どこを見ればよいか？

エラーは「敵」ではなく「ヒント」

🔴 コンパイルエラー

実行 前 に発覚
文法の間違い

例：End If が抜けている

🟠 実行時エラー

実行 中 に発生
エラー番号が画面に出る

例：存在しないファイルを開こうとした

🟡 論理エラー

エラーは出ないが
結果が間違い

例：列番号が1つずれていた

※ エラー番号はそのまま Google で検索できます（例：「VBA エラー 1004」）

武器① Debug.Print

Sub 売上計算()
    Dim i As Long
    For i = 1 To 10
        ' ↓ イミディエイトウィンドウに変数の値を出力する
        Debug.Print "行:" & i & "  商品名:" & Cells(i, 1).Value
    Next i
End Sub

使い方

  • Ctrl + G でウィンドウを開く
  • 処理が 止まらない ので
    ループ内の確認に最適

MsgBox との違い

  • MsgBox：確認するたびクリックが必要
  • Debug.Print：一気に流れを確認できる

武器② ブレークポイント F9

操作手順

  • 止めたい行の左端をクリック → ● が表示される
  • F5 でマクロを実行 → その行で一時停止
  • 変数の上にマウスオーバー → 現在値 が表示
  • F9 でブレークポイントを解除

注意点

  • ブックを閉じると
    ブレークポイントは消える
  • 「怪しい箇所の直前」
    に設定するのがコツ

武器③ ステップ実行 F8

📌 F8 を押すたびに1行ずつ実行される

コードの動きを目で追いながら、どこで問題が起きているかを特定できる

よくある使い方

  • コードを書いたら
    まず F8 で実行してみる
  • ループが想定通りに
    回っているか確認する

組み合わせ技

  • ブレークポイントで止める
  • そこから F8 で
    1行ずつ確認していく

02

エラーハンドリング

マクロが途中で止まっても、ちゃんと終われるようにするには？

エラー処理がないと何が起きるか

❌ エラー処理なし

  • マクロが途中で止まる
  • ScreenUpdating が False のまま
    → 画面が更新されなくなる
  • 開きっぱなしのブックが残る
  • ユーザーが「何が起きたかわからない」状態になる

✅ エラー処理あり

  • 何が起きたかメッセージで伝える
  • 後片付けを確実に行う
  • 問題ファイルをスキップして続行

On Error GoTo の構造

Sub サンプル()

    On Error GoTo ErrorHandler   ' ① エラーが起きたらここへ飛ぶと宣言

    '── 通常の処理 ──
    Workbooks.Open "C:\data.xlsx"

    Exit Sub                     ' ② 正常終了（←これがないと ErrorHandler も動く！）

ErrorHandler:                    ' ③ エラー発生時の処理
    MsgBox "エラー番号：" & Err.Number & vbCrLf _
         & "内容："       & Err.Description
    Application.ScreenUpdating = True   ' ④ 後片付け

End Sub
⚠️ よくある落とし穴：Exit Sub を忘れると、正常終了時も ErrorHandler が実行されてしまいます

Err オブジェクトで詳細を取得する

主なプロパティ

  • Err.Number
    エラー番号（整数）
  • Err.Description
    エラーの説明文

よく出るエラー番号

  • 53　ファイルが見つからない
  • 1004　オブジェクトエラー
  • 9　インデックスが範囲外
  • 13　型が一致しない

エラー番号はそのまま Google 検索に使えます（例：「VBA エラー 53」）

On Error Resume Next の使いどころ

✅ 使ってよいケース

' シートの存在確認
On Error Resume Next
Set ws = Worksheets("シート名")
On Error GoTo 0  ' ← 必ずリセット！

If ws Is Nothing Then
    MsgBox "シートがありません"
End If

❌ 使ってはいけないケース

' ループ全体に使う（危険）
On Error Resume Next
For i = 2 To lastRow
    ' エラーが出ても
    ' 全部スルーされてしまう
Next i
On Error GoTo 0

03

Function（関数）

同じ処理を何度も書かずに済む方法

「同じ処理が3回ある」問題

Sub 集計()
    ' ここでも…
    lastRow1 = Cells(Rows.Count, 1).End(xlUp).Row  ' ← 同じ処理

    ' またここでも…
    lastRow2 = Cells(Rows.Count, 2).End(xlUp).Row  ' ← 同じ処理

    ' またまたここでも…
    lastRow3 = Cells(Rows.Count, 3).End(xlUp).Row  ' ← 同じ処理
End Sub
同じコードが複数箇所にある → 修正漏れが発生しやすい → バグの原因になる

Sub と Function の違い

SubFunction
戻り値なしあり
呼び出し方Call プロシージャ名変数 = 関数名(引数)
主な用途一連の処理を実行する計算・判定結果を返す
Function 最終行取得(ws As Worksheet, col As Long) As Long
    '                ↑ 引数                         ↑ 戻り値の型
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
    '  ↑ 関数名 = 戻り値  という形で返す
End Function

Function を使った Before / After

Before（重複コード）

Sub 集計()
  lastRow1 = Cells(
    Rows.Count, 1
  ).End(xlUp).Row
  lastRow2 = Cells(
    Rows.Count, 2
  ).End(xlUp).Row
  lastRow3 = Cells(
    Rows.Count, 3
  ).End(xlUp).Row
End Sub

After（Function に切り出す）

Function 最終行取得(
  ws As Worksheet,
  col As Long) As Long
  最終行取得 = ws.Cells(
    ws.Rows.Count, col
  ).End(xlUp).Row
End Function

Sub 集計()
  lastRow1 = 最終行取得(ws, 1)
  lastRow2 = 最終行取得(ws, 2)
  lastRow3 = 最終行取得(ws, 3)
End Sub

引数の渡し方：ByVal / ByRef

ByVal（基本はこちら）

値のコピーを渡す
元の変数は変わらない

Function 計算(ByVal 数値 As Long)

ByRef（意図して使う場合のみ）

参照を渡す
関数の中で元の変数も変わる

Function 計算(ByRef 数値 As Long)

💡 迷ったら ByVal を使う

04

シートの高度な操作

シートを動的に増やしたり、切り替えながら処理するには？

シートの参照方法（安全な書き方）

' NG：ActiveSheet はどのシートが選ばれているか保証がない
ActiveSheet.Cells(1, 1).Value = "タイトル"

' NG：シート名が変わると動かなくなる
Worksheets("集計").Cells(1, 1).Value = "タイトル"

' ✅ OK：変数に入れてから使う ← この書き方を習慣に
Dim ws As Worksheet
Set ws = ThisWorkbook.Worksheets("集計")  ' Set が必要！
ws.Cells(1, 1).Value = "タイトル"
⚠️ オブジェクト変数の代入には Set が必要です（忘れるとエラーになります）

シートの作成・削除

' ① 既存シートの確認と削除
Application.DisplayAlerts = False   ' 削除時の確認ダイアログを非表示
Dim ws As Worksheet
On Error Resume Next
Set ws = Worksheets("集計")
On Error GoTo 0

If Not ws Is Nothing Then
    ws.Delete
    Set ws = Nothing
End If

' ② 新しいシートを末尾に追加
Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = "集計"

Application.DisplayAlerts = True    ' 必ず元に戻す

For Each で全シートをループする

' 全シートをループ
Dim ws As Worksheet
For Each ws In ThisWorkbook.Worksheets
    Debug.Print ws.Name   ' 各シートの名前を出力
Next ws

' 特定のシートをスキップしたい場合
For Each ws In ThisWorkbook.Worksheets
    If ws.Name = "演習説明" Then GoTo NextSheet   ' スキップ
    ' 処理
NextSheet:
Next ws

💡 For Each は「いくつあるかわからない」コレクションをループするときに使う

パフォーマンス最適化の定番コード

Sub メインマクロ()

    ' ─── 処理前：停止する ───
    Application.ScreenUpdating  = False            ' 画面更新を止める
    Application.Calculation     = xlCalculationManual  ' 自動計算を止める
    Application.DisplayAlerts   = False            ' 確認ダイアログを非表示

    On Error GoTo ErrorHandler

    ' ─── 実際の処理 ───

    Call 後片付け
    MsgBox "完了しました", vbInformation
    Exit Sub

ErrorHandler:
    MsgBox "エラー：" & Err.Description, vbCritical
    Call 後片付け
End Sub

Sub 後片付け()
    Application.ScreenUpdating  = True
    Application.Calculation     = xlCalculationAutomatic
    Application.DisplayAlerts   = True
End Sub
