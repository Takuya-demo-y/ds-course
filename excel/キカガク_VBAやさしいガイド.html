基本のかたち 1日目
VBAコードを書く「箱」の作り方。まずここから始めましょう。
💬 ちょっと雑談
みなさん、VBAを使う前は毎月の集計作業をどうやっていましたか？
「同じファイルを何十回もコピペして…」「合計欄を手で直して…」という方、けっこういますよね。
実はそれ、今日学ぶたった数行のコードで自動化できます。「なんでもっと早く知らなかったんだ！」という声、毎回聞きます笑
焦らずいきましょう。まず VBA の「コードを書く箱」の作り方から始めます。
📖 Sub は「レシピカード」だと思ってください。
カードには名前があって、中に手順が書いてある。VBAも同じです。Sub 名前() がカードの表紙、End Sub が裏表紙。その間に「何をするか」を書くだけ。
1
Sub プロシージャ名() と書く
これが「箱の名前と入口」。ここからコードが始まります。名前は日本語でもOKです。
2
中に処理を書く
インデント（字下げ）を使うと読みやすくなります。VBEは自動でつけてくれます。
3
End Sub で閉じる
これが「箱の出口」。コツは最初に Sub ... と End Sub を両方書いて、中身を後から埋めること。

📊 Sub と Function の違い

🏃
Sub
仕事をこなすだけ
戻り値なし ボタンから実行できる メインの処理に使う
VS
🔍
Function
仕事して、答えを返す
戻り値あり 他のコードから呼び出す 何度でも使い回せる 
' Sub：処理をまとめる入れ物。ボタンから実行できる
Sub 月次集計()
    MsgBox "集計が完了しました"
End Sub

' Function：「答えを返せる」特別な入れ物
Function 売上評価(金額 As Long) As String
    If 金額 >= 1000000 Then
        売上評価 = "A"
    Else
        売上評価 = "B"
    End If
End Function
🤔 Sub と Function、どう使い分けるの？
シンプルな判断基準はこれです：「答えを返してほしいか？」
「最終行は何行目？」「この金額は何評価？」→ 答えが必要 → Function
「ファイルを開いて転記して閉じる」→ ただ仕事するだけ → Sub
🏢 実務での使い方：
「毎月の集計ボタン」→ Sub で書いてボタンに登録。「最終行を調べる」「売上評価を計算する」→ Function に切り出して使い回す。この組み合わせが業務マクロの基本パターンです。

変数って何？ 1日目
値を一時的に保存する「名前のついた箱」を作る
💬 ちょっと雑談
「変数」という言葉、数学の x や y を思い出して「難しそう…」と感じた方、いますか？
大丈夫です。プログラミングの変数は「名前のついた箱」——それだけです。
たとえば「今月の最終行は何行目か」という数字を覚えておきたいとき、その数字をどこかに保存しておかないといけないですよね。そのための箱が変数です。
ちなみに変数名をつけるのって、意外と楽しいんですよ。センスが出ます笑
📦 変数は「ラベルを貼った引き出し」です。
「整数を入れる i という引き出し」→ Dim i As Long
「文字列を入れる name という引き出し」→ Dim name As String
一度作った引き出しには何度でも違う値を入れ替えられます。
1
Dim 変数名 As 型 と書く（宣言）
「この名前の箱を用意してください」という宣言です。Sub の先頭にまとめて書くのが慣習です。
2
変数名 = 値 で値を入れる（代入）
箱に値を入れます。同じ箱に何度でも入れ替えられます。
3
変数名を書くと中の値が使える（参照）
Cells(i, 1) のように書けば、i の中の数字がそのまま使われます。

ボタンを押すと変数に値が入ります

📋 よく使う型の一覧

🔢
Long（整数）
ループのカウンター・行番号など整数全般に使う。最も頻出。
Dim i As Long
📝
String（文字列）
名前・ファイル名・評価など文字列に使う。ダブルクォートで代入。
Dim name As String
🔄
Boolean（真偽値）
True か False の2値だけ。「処理したか否か」などのフラグに使う。
Dim flag As Boolean 
Dim i       As Long      ' 整数（ループのカウンターに）
Dim name    As String    ' 文字列
Dim price   As Double    ' 小数点あり数値
Dim flag    As Boolean   ' True / False
Dim ws      As Worksheet ' シートを入れる箱 ← Set が必要！
Const FOLDER As String = "C:\data\"  ' 定数（変わらない値）

i = 5
name = "田中"
Set ws = Worksheets("集計")    ' オブジェクト型には Set が必要！
😅 ここでよくつまずきます：Worksheet には Set が必要！
普通の変数は i = 5 と書けばOKですが、Worksheet や Workbook のような「オブジェクト型」は Set ws = Worksheets("集計") のように Set をつけます。
忘れると「オブジェクト変数が設定されていません」というエラーが出ます。「またこのエラーか」→ まず Set を確認。

セルを触る 1日目
Excelのマス目を読んだり書いたりする2つの方法
💬 ちょっと雑談
Excelって、実はすべてのセルが「行番号・列番号」の座標で管理されているんです。
A1 とか B2 という書き方に慣れすぎていて気づかないけど、A列は1列目、B列は2列目… というルールがあるんですよ。
だから VBA では Cells(1, 1) って書くと A1 のことになります。「え、A1 って書けないの？」と思うかもしれませんが、実は両方書けます。使い分けのコツを見ていきましょう！
🗺️ セルは「座標で呼ぶ」と覚えてください。
地図で「北緯35度・東経139度」と指定するのと同じ感覚。Cells(2, 3) は「2行目・3列目」= C2セルです。
1
Cells(行, 列) ── ループで使いやすい
行と列を数字で指定。For i = 2 To lastRow のように変数を使えるので、ループの中で行を変えながら処理できる。
2
Range("A1") ── 固定セルを名前で指定
「タイトルは常にA1」など固定のセルを指定するとき。コードが読みやすくなる。
3
.Value で読み書き
書き込みは Cells(2,1).Value = "田中"、読み込みは name = Cells(2,1).Value。

📊 Cells(行, 列) の座標イメージ

A列(1)B列(2)C列(3)D列(4)
1行商品名数量単価金額
2行りんご3 ★ 150
Cells(2,3)		450
3行みかん580 計算中
Cells(i,4)

🟡 黄色 = Cells(2,3)（C2セル） 🔵 青 = ループで処理中のセル

Cells(2, 3)                          ' 2行3列 = C2セル
Range("A1")                         ' 名前で指定

Cells(2, 1).Value = "田中"         ' A2に書き込む
name = Cells(2, 1).Value            ' A2の値を読み込む

' ★ 最終行の取得（必ず覚えたい定番パターン）
lastRow = Cells(Rows.Count, 1).End(xlUp).Row
🤔 Cells(Rows.Count, 1).End(xlUp).Row ── 意味がわからない…
分解して読みましょう：
① Cells(Rows.Count, 1) → A列の「一番最後の行」（約100万行目）
② .End(xlUp) → そこから上に向かって Ctrl+↑ を押した結果のセル
③ .Row → そのセルの行番号を取得
Excelで実際に Ctrl+↑ を押してみると、同じ動きをしますよ。
🏢 実務での使い方：
毎月行数が変わる集計データに最適。For i = 2 To lastRow と書けば、データが100行でも1000行でも同じコードが動きます。

もし〜ならば（If） 1日目
条件によって処理を変える。業務マクロの核心です。
💬 ちょっと雑談
みなさんって、無意識のうちに一日に何千回も「もし〜ならば」という判断をしていると思うんですよ。
「もし雨なら傘を持って行こう」「もし残業なら外で夕食にしよう」「もし打ち合わせが長引いたら電車を1本遅らせよう」…
人間の思考回路って、ほぼ全部 If 文でできてるんです。笑
プログラムも同じ。「もし売上が100万以上ならA評価」「もしファイルがあれば開く」——これをコードで書けると、一気にできることが増えます。
🚦 信号と同じです。
「青なら進む、赤なら止まる、黄なら気をつけて進む」——それをコードで書いたのが If 文です。

📊 If 文の流れ（売上評価の例）

売上金額を受け取る
↓
売上 ≥ 100万円？
Yes
↓
"A評価" を返す
No
↓
売上 ≥ 50万円？
Yes
↓
"B評価"
No
↓
"C評価"
↓
評価をセルに書き込む
If     売上金額 >= 1000000 Then
    評価 = "A"
ElseIf 売上金額 >= 500000  Then
    評価 = "B"
Else
    評価 = "C"
End If

' 複数条件：AかつB
If A > 0 And B > 0 Then  ...
' 複数条件：AまたはB
If A > 0 Or  B > 0 Then  ...
⚠️ 必ず「End If」で閉じること！
If を書いたら最後に End If が必要です。コツは最初に If ... Then と End If を両方書いてしまって、中身を後から埋めること。
🏢 実務での使い方：
「売上が100万以上ならA評価」「ファイルが存在するなら開く、なければスキップ」「シートが集計ならば処理、それ以外はスキップ」——業務マクロの半分以上は If 文で成り立っています。

繰り返す（For / Do While） 1日目
「同じ作業を何百回も繰り返す」という仕事を自動化する
💬 ちょっと雑談
ループを初めて書いて実行したとき、どんな感覚でしたか？
「え、これだけで全部やってくれるの！？」って、ちょっと感動しませんでしたか？笑
ループを覚えた瞬間、「今まであの作業は何だったんだ…」と少し悲しくなる方もいます。1000行のデータを手で処理していた時間を返してほしい、と。
でも、今日からはその必要はありません。ループに全部任せましょう！
🏭 工場のベルトコンベアのイメージです。
商品がひとつずつ流れてきて、同じ検査・加工を繰り返す。VBAのループも「2行目から最終行まで同じ処理を繰り返す」動きをします。人が2時間かかる作業が数秒で終わります。

📊 3種類のループの使い分け

🔢
For〜Next
「何回繰り返すか」あらかじめわかっているとき。2行目から最終行まで、など。
範囲がわかる
📂
Do While〜Loop
「いつ終わるかわからない」とき。ファイルがある限り続ける、など。
条件が続く間
📋
For Each〜Next
全シート・全ファイルなど「コレクションの全部」に対して処理したいとき。
全部に対して
1
For i = 2 To lastRow と書く
i は 2 から始まり、lastRow まで 1 ずつ増えながら繰り返す。
2
処理を書く（Cells(i, ...) で行を変えながら）
i が変わるたびに、Cells(i, 4) も次の行を指す。
3
Next i で閉じる
ここで i が 1 増えて、ループの先頭に戻る。
Dim i As Long
For i = 2 To lastRow
    Cells(i, 4).Value = Cells(i, 2).Value * Cells(i, 3).Value
Next i
🤔 「なんで i = 1 じゃなくて i = 2 から始めるの？」
1行目はヘッダー行（「商品名」「数量」などの見出し）であることが多いので、データのある2行目から処理を始めます。ヘッダー行を計算してしまうとエラーや変な結果になります。

Dir() ── 「フォルダのファイルを1つずつ取り出す」

🎮 インタラクティブ：Dir() の動きを確認

📄 東京.xlsx
📄 大阪.xlsx
📄 名古屋.xlsx
Dir() の戻り値：
— まだ実行していません — 
fileName = Dir("C:\data\*.xlsx")   ' 最初だけ引数あり
Do While fileName <> ""
    ' 処理...
    fileName = Dir()               ' 次のファイルへ（引数なし）← 必須！
Loop
😅 Dir() の動きが直感に反しますよね。
最初の Dir("*.xlsx") は「最初のファイル名を返す」のに、2回目以降は Dir() と引数なしで「次のファイル名を返す」。「最初だけ引数あり、次からは引数なし」と丸暗記してOKです。
⚠️ 無限ループに注意！
fileName = Dir() を書き忘れると、同じファイルをずっと処理し続けてExcelが固まります。Esc キーで強制停止できます。

ファイルとシートを操る 1日目
ブックを開いて処理して閉じる。シートを追加・削除する
💬 ちょっと雑談
ファイルを開きっぱなしにしてPCが重くなった経験、ありませんか？
VBAでも同じです。開いたファイルをずっと放置するとメモリを圧迫します。
「開いたら必ず閉じる」——これだけで多くのトラブルが防げます。
あと、ファイルパスを絶対パス（C:¥Users¥yamada¥...）で書いてしまうと、自分のPCでしか動かなくなります。チームで使うツールを作るときは要注意です！
📚 本棚から本を取り出して、読んで、棚に戻すイメージです。
開く→処理する→閉じる。この3ステップを守れば、ファイル操作は怖くありません。 
' ★ ThisWorkbook.Path を使う（絶対パスより安全）
path = ThisWorkbook.Path & ""

' 開く前にファイルの存在を確認（安全！）
If Dir(path & "データ.xlsx") <> "" Then
    Workbooks.Open path & "データ.xlsx"
    Set wb = ActiveWorkbook
    Set ws = wb.Worksheets("Sheet1")
    ' 処理 ...
    wb.Close SaveChanges:=True   ' ← 開いたら必ず閉じる！
End If

' シートを削除（確認ダイアログを消す）
Application.DisplayAlerts = False
Worksheets("削除するシート").Delete
Application.DisplayAlerts = True  ' ← 必ず戻す！
🤔 ThisWorkbook.Path と絶対パス、どちらを使えばいい？
絶対パスで書くと「自分のPCでしか動かないツール」になります。
ThisWorkbook.Path を使えば「どのPCに置いても動くツール」になります。チームで使うなら必ず ThisWorkbook.Path を使いましょう。

デバッグ ── バグを探す武器 2日目
コードが思い通りに動かないとき、どこを見ればいい？
💬 ちょっと雑談
バグが出た時、みなさんはどう対処していますか？
「もうわからん！」ってコードを全部消したり、PCを叩きたくなったりする気持ち…わかります笑
でも実は、バグには「必ず原因がある」んです。ランダムに壊れることはありません。
道具の使い方を知っていれば、ほとんどのバグは30分以内に見つかります。
エラーは「敵」ではなく「ヒント」です。 冷静に一緒に追いかけましょう！
🍳 レシピ通りに作ったのに料理がおいしくない。
材料も揃っている、手順も合っている、でも何かがおかしい。「全部捨てて最初からやり直す」はもったいない。「どこが間違っているか冷静に確認する」——それがデバッグです。

🛠️ デバッグ 3 つの武器

Ctrl + G
① Debug.Print
値をイミディエイトウィンドウに出力。止まらないのでループの中でも全部記録できる。
大量の値を記録したい
F9 → F5
② ブレークポイント
行の左端クリックで赤丸●。F5実行するとその行でピタッと停止。変数をマウスオーバーで値確認。
ここで何が起きてる？
F8
③ ステップ実行
F8を押すたびに1行ずつ進む。黄色ハイライトを目で追いながら変数の変化を確認。
どこがおかしいか追う
1
まず Debug.Print でループの値を全部出力してみる
Ctrl+G でイミディエイトウィンドウを開いて確認。「ここで変な値になってる！」と気づける。
2
怪しい行にブレークポイント（F9）を置く
行の左端をクリックすると赤丸●が付く。F5で実行するとそこで止まり、変数の今の値をマウスオーバーで確認できる。
3
どこが怪しいかまったくわからない → F8 で最初からゆっくり
1行ずつ実行して黄色ハイライトを追う。時間はかかるが確実。
' ── 武器① Debug.Print ─────────────────────
Debug.Print "行:" & i & "  値:" & Cells(i, 1).Value

' ── 武器②③ はキーボード操作 ──────────────
' F9 でブレークポイント設定 → F5で実行して停止
' F8 でステップ実行（1行ずつ進む）
🤔 「MsgBoxで確認すればよくない？」と思いますよね。
MsgBox は毎回「OK」を押さないといけないので、100回ループする処理では100回クリックが必要になります。Debug.Print はクリック不要で全部流れていくので、ループの中の確認には断然 Debug.Print の方が楽です。

📋 3種類のエラーと見つけ方

🔴
コンパイルエラー
実行前に検出される（文法ミス）
VBEが赤くハイライト。End If の閉じ忘れ・括弧ミスが多い。一番わかりやすいエラー。
🟠
実行時エラー
実行中に突然ダイアログが出る
エラー番号をそのままGoogleで検索すると解決策がすぐ見つかる。「型が一致しません」「インデックスが有効範囲にありません」が頻出。
🟡
論理エラー（一番厄介！）
実行は成功するが結果が間違っている
VBAは一切教えてくれない。結果を見て初めて気づく。「合計が合わない」→ Debug.Print で変数を1つずつ追う。
🛡
エラーを「防ぐ」 2日目
エラーが起きてから対処するより、起きないようにする
💬 ちょっと雑談
「マクロを実行したら急に止まった！」という経験、ありませんか？
そういうとき、たいていの原因はこれです：「ファイルがない」「シートがない」「想定外のデータが入っていた」
実務では、想定外のことが普通に起きます。ファイルが届いていない月、誰かがシートを削除してしまった…など。
「エラーが起きてから直す」より「エラーが起きないようにする」方が、ずっとスマートです。しかも、1日目で学んだ If と Dir だけで書けますよ！
🚪 ドアを勢いよく開けてぶつかるより、ノックしてから開ける方が安全です。
ファイルを開くときも同じ。「このファイル、ちゃんとありますか？」と確認してから開く方が親切で安全なコードになります。

📊 安全な開き方 vs 危険な開き方

❌ 危険な開き方
1
Workbooks.Open filePath を即実行
↓
2
ファイルが存在しない場合…
↓
💥 実行時エラー！マクロが止まる
✅ 安全な開き方
1
Dir(filePath) <> "" でファイル確認
↓
2
あれば → 開いて処理
なければ → メッセージ+スキップ
↓
✅ エラーにならず次の処理へ
' ✅ ファイル確認してから開く
If Dir(filePath) <> "" Then
    Workbooks.Open filePath
Else
    MsgBox filePath & " が見つかりません"
End If

' ✅ シートの存在確認
On Error Resume Next             ' ← ここだけエラーを無視
Set ws = Worksheets("集計")
On Error GoTo 0                  ' ← すぐリセット（この2行はセット！）
If Not ws Is Nothing Then
    ' シートが存在する場合の処理
End If
😅 On Error Resume Next ── 怖くないですか？
「エラーを無視する」と聞くと不安になりますよね。
大事なのは直後に必ず On Error GoTo 0 でリセットすること。この2行は常にセットで書く、と覚えてください。

Function ── 答えを返す関数を作る 2日目
同じ処理を1か所にまとめて何度でも使い回す
💬 ちょっと雑談
コードを書いていて「あれ、同じような処理を2回書いてる…」と感じた経験、ありませんか？
そのとき心のどこかで「なんかスッキリしないな」と感じませんでしたか？
その感覚、正解です！
プログラマーの世界に「DRY原則」という言葉があります。「Don't Repeat Yourself」——同じコードを繰り返すな、という意味です。
Function を使って「共通の処理を1か所にまとめる」——これだけで修正が圧倒的に楽になります。
🏪 コンビニのレジマニュアルを作るイメージです。
修正が必要になったときもマニュアルを1か所直すだけで全員に反映されます。Function も同じ——処理を1か所にまとめておけば、修正はいつも1か所で済みます。

🔄 Before / After を切り替えて比較

Sub 月次集計() ' 同じコードが3か所に散らばっている！ row1 = ws1.Cells(Rows.Count,1).End(xlUp).Row ' ← 同じ row2 = ws2.Cells(Rows.Count,1).End(xlUp).Row ' ← 同じ row3 = ws3.Cells(Rows.Count,1).End(xlUp).Row ' ← 同じ ' ⚠ 修正が必要になったら3か所全部直す必要がある End Sub
Function 最終行取得(ws As Worksheet, col As Long) As Long 最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row ' ✅ 修正はここだけ！全部に自動で反映される End Function Sub 月次集計() row1 = 最終行取得(ws1, 1) ' ← シンプル！ row2 = 最終行取得(ws2, 1) row3 = 最終行取得(ws3, 1) End Sub
1
Function 名前(引数 As 型) As 戻り値の型 と書く
「この関数は〇〇を受け取って△△を返す」という宣言。
2
処理を書いて 関数名 = 戻り値 で答えを返す
他の言語の return 値 に相当する書き方。End Function の前に書かないと 0 や空文字が返ってしまう。
3
別の Sub から result = 関数名(引数) で呼び出す
どこからでも何度でも呼び出せる。
' ── 定義 ─────────────────────────────────────
Function 最終行取得(ws As Worksheet, col As Long) As Long
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
End Function

' ── 呼び出し ──────────────────────────────────
lastRow = 最終行取得(ActiveSheet, 1)
grade   = 売上評価(Cells(i, 4).Value)
😅 「関数名 = 値」という書き方、最初は不思議ですよね。
他の言語では return 値 と書きますが、VBAでは 関数名 = 戻り値 と書きます。「この変数（関数名）に答えを入れておくよ」というイメージで覚えると少し納得できます。

💬 ちょっと雑談
マクロを実行したら画面がチカチカして…Excelが固まった！？という経験、ありませんか？笑
あれ、実は VBA が処理のたびに画面を更新しているせいなんです。100ファイル処理するなら、100回画面が更新される。それが遅さの原因です。
ScreenUpdating を False にするだけで、10倍以上速くなることもあります。
「たったこれだけで？」と思いますよね。でも本当にこれだけです。やってみてください！
🍳 「料理中に都度、鍋の中を見せてもらえる？」と言われたら…
毎回手を止めて見せるのは大変ですよね。VBAも処理のたびに画面を更新するので遅くなります。ScreenUpdating = False は「全部終わったら一気に見せます」という宣言です。

💬 ちょっと雑談
業務データをVBAで処理していると、必ずと言っていいほどぶつかる壁があります。
「セルから読んだ文字に、なぜか空白が混ざっていて一致しない！」
「数値のはずなのに文字列として扱われて計算できない！」
どちらも今日紹介する関数1つで解決できます。 Trim と型変換、特に覚えておいてください。これで実務の「謎のエラー」がかなり減ります笑
💬 Trim は「空白を除く消しゴム」です。
Excelのセルにスペースが混入していて「なぜか一致しない！」というトラブル、よくあります。Trim(Cells(i,1).Value) で囲むだけで解決することが多いです。

🎮 インタラクティブ：文字列操作の見え方

Len("Hello")            ' 文字数      → 5
Left("Hello", 3)        ' 左から3文字 → "Hel"
Right("Hello", 3)       ' 右から3文字 → "llo"
Mid("Hello", 2, 3)      ' 2文字目から3文字 → "ell"
Trim("  Hello  ")       ' 前後の空白除去 → "Hello"

CLng("123")   ' 文字列 → Long
CStr(123)    ' 数値 → 文字列
Year(Now())   ' 現在の年
Month(Now())  ' 現在の月
🤔 「型変換って何のためにするの？」
Excelのセルから読み込んだ「123」には2種類あります。数値の 123（右寄せ）と文字列の "123"（左寄せ）。文字列のまま計算しようとするとエラーや変な結果になります。「なんかおかしい」と思ったら型変換を試してみてください。
