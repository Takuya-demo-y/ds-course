### スライド 03｜エラーは「敵」ではなく「ヒント」

```
よくある反応：
  エラーが出た → パニック → 全部消す → また書く → また出る

理想的な反応：
  エラーが出た → 番号とメッセージを読む → 場所を特定する → 修正する
```

**▶ 話すこと**
- エラーダイアログの「デバッグ」ボタンを押すと、エラーが起きた行がハイライトされる。
- エラー番号はそのままGoogleで検索できる（「VBA エラー 1004」など）。
- 初心者のうちは「エラーが出た数だけ成長している」と伝えて心理的ハードルを下げる。

---

### スライド 04｜エラーの3種類

| 種類 | タイミング | 例 |
|---|---|---|
| コンパイルエラー | 実行前 | `End If` が抜けている |
| 実行時エラー | 実行中 | 存在しないファイルを開こうとした |
| 論理エラー | エラーは出ない | 計算式の列番号が1つずれていた |

**▶ 話すこと**
- 最も厄介なのは「論理エラー」。VBAは正しく動いているのに、結果が間違っている。
- デバッグ技術が必要になるのは、主にこの論理エラーへの対処。
- 今日の演習1はまさにこれ。

### スライド 05｜武器1：`Debug.Print`

```vba
Sub 売上計算()
    Dim i As Long
    For i = 1 To 10
        Debug.Print "i の値：" & i & "  商品名：" & Cells(i, 1).Value
        '                ↑ ここでループの中身を可視化する
    Next i
End Sub
```

**実演手順**
1. VBE を開く（`Alt + F11`）
2. `Ctrl + G` でイミディエイトウィンドウを開く
3. 上のコードを実行してウィンドウに出力されることを確認

**▶ 話すこと**
- `MsgBox` との違いを強調：処理が止まらないので、ループ内での確認に最適。
- 確認が終わったら `'` でコメントアウトしておく習慣を伝える。

### スライド 06｜武器2・3：ブレークポイントとステップ実行

```
操作手順：

1. 止めたい行の行番号の左をクリック
   → 赤い丸（●）が表示される ← これがブレークポイント

2. F5 でマクロを実行
   → 設定した行で処理が一時停止する

3. マウスを変数の上に乗せる
   → 現在の値がツールチップで表示される

4. F8 を押すと1行ずつ進む
   → コードの動きを目で追える
```

**▶ 話すこと**
- 「コードを書いたらまず `F8` で動かす」習慣を強調する。
- ブレークポイントは複数設定できる。複雑な処理は「怪しい箇所の直前」に設定する。

**⚠ 注意**
- ブレークポイントはブックを閉じると消えてしまう。保存はされない。

### スライド 07｜エラー処理なしのコードの問題

```
【問題シナリオ】
  東京・大阪・名古屋の支店ファイルを順番に開いて集計するマクロを実行中…

  処理の途中で「名古屋支店.xlsx が見つかりません」エラーが発生！

  結果：
  ・マクロが途中で止まる
  ・ScreenUpdating が False のままになり、画面が更新されなくなる
  ・開きっぱなしのブックが残る
  ・ユーザーは「何が起きたかわからない」状態になる
```

**▶ 話すこと**
- エラー処理は「もしものための保険」。あって損はない。
- 特に「他の人が使うマクロ」には必ず入れるべき。
- `ScreenUpdating` などの後片付けを確実に行うための仕組みでもある。

### スライド 08｜`On Error GoTo` の構造（図解）

```
Sub サンプル()
    │
    ├── On Error GoTo ErrorHandler  ─── ① 「エラーが起きたら下のラベルへ」と宣言
    │
    ├── 処理A
    ├── 処理B  ←──────── ここでエラー発生！
    ├── 処理C               │
    │                       │ ジャンプ
    ├── Exit Sub ───────────┼─── ② 正常終了（ErrorHandler をスキップ）
    │                       │
    └── ErrorHandler:  ◄────┘  ③ エラー処理
            MsgBox Err.Description
```

**▶ 話すこと**
- `Exit Sub` を入れる理由を必ず説明する（これを忘れる受講者が多い）。
- 「正常に終わったときも ErrorHandler に入ってしまう」という落とし穴を実演する。

**⚠ よくある間違い**
```vba
' NG：Exit Sub がないと正常終了時も ErrorHandler が実行される
Sub NG例()
    On Error GoTo ErrorHandler
    MsgBox "正常処理"
    ' Exit Sub がない！

ErrorHandler:
    MsgBox "これが正常終了時にも表示されてしまう"
End Sub
```

### スライド 09｜`Err` オブジェクトの使い方

```vba
ErrorHandler:
    MsgBox "エラーが発生しました" & vbCrLf & _
           "番号：" & Err.Number      & vbCrLf & _
           "内容：" & Err.Description

    ' 後片付けを忘れずに
    Application.ScreenUpdating = True
```

**▶ 話すこと**
- `Err.Number` は整数。よく出るエラー番号をいくつか紹介する。
  - `1004`：オブジェクトエラー（存在しないシートへのアクセスなど）
  - `9`：インデックスが範囲外（存在しない配列の添字など）
  - `53`：ファイルが見つからない
- エラー番号をログに残しておくと、後から原因調査がしやすい。

### スライド 10｜`On Error Resume Next` の使い方と注意点

```vba
' 良い例：単発の存在確認にだけ使う
On Error Resume Next
Set ws = Worksheets("確認するシート名")
On Error GoTo 0      ' ← 必ずここでリセットする！

If ws Is Nothing Then
    MsgBox "シートが見つかりません"
End If

' 悪い例：範囲が広すぎる
On Error Resume Next    ' ← ここから
  処理A
  処理B   ← ここでエラーが出ても無視される
  処理C
On Error GoTo 0         ' ← ここまで全部無視
```

**▶ 話すこと**
- `On Error Resume Next` は「エラーを見逃す」道具。乱用は禁物。
- 使い終わったら必ず `On Error GoTo 0` でリセットすることを徹底する。

### スライド 11｜「同じ処理が3回ある」コードの問題

```vba
Sub 集計()
    ' ここでも最終行を取得
    lastRow1 = Cells(Rows.Count, 1).End(xlUp).Row

    ' 〜 処理 〜

    ' またここでも最終行を取得
    lastRow2 = Cells(Rows.Count, 2).End(xlUp).Row

    ' 〜 処理 〜

    ' またまたここでも最終行を取得
    lastRow3 = Cells(Rows.Count, 3).End(xlUp).Row
End Sub
```

**▶ 話すこと**
- 同じコードが複数箇所にあると、修正するときに「全部変えないといけない」。
- バグを直したつもりが、直していない箇所が残る → 見つけにくいバグの原因になる。
- 「1つの処理は1か所に書く（DRY原則）」を伝える。

---

### スライド 12｜Sub と Function の違い

```vba
' Sub：何かする（結果は返さない）
Sub 画面を更新する()
    Application.ScreenUpdating = True
End Sub

' Function：何かして結果を返す
Function 最終行取得(ws As Worksheet, col As Long) As Long
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
End Function

' 呼び出し方の違い
Call 画面を更新する          ' Sub は Call で呼ぶ（Call は省略可）
lastRow = 最終行取得(ws, 1)  ' Function は変数に受け取る
```

**▶ 話すこと**
- 「`関数名 = 値`」という書き方が最初は不思議に感じると思う。「関数名という変数に代入する」イメージと伝える。
- Function を使わなくてもマクロは動く。でも使うと「読みやすく、直しやすいコード」になる。

---

### スライド 13｜Function を使った Before / After

```vba
' ─── Before（Function なし）───
Sub 集計()
    lastRow1 = Cells(Rows.Count, 1).End(xlUp).Row
    lastRow2 = Cells(Rows.Count, 2).End(xlUp).Row
    lastRow3 = Cells(Rows.Count, 3).End(xlUp).Row
End Sub

' ─── After（Function あり）───
Function 最終行取得(ws As Worksheet, col As Long) As Long
    最終行取得 = ws.Cells(ws.Rows.Count, col).End(xlUp).Row
End Function

Sub 集計()
    lastRow1 = 最終行取得(ws, 1)   ' ← 意図が分かりやすい！
    lastRow2 = 最終行取得(ws, 2)
    lastRow3 = 最終行取得(ws, 3)
End Sub
```

**▶ 話すこと**
- コードの「読みやすさ」が保守コストに直結する。
- 半年後に自分のコードを読んだときに意味がわかるかどうかが判断基準。

### スライド 14｜引数の渡し方（ByVal / ByRef）

```vba
' ByVal：コピーを渡す（元は変わらない）← 基本はこちら
Function 2倍にする(ByVal 数値 As Long) As Long
    数値 = 数値 * 2   ' コピーを変えているので元の変数は変わらない
    2倍にする = 数値
End Function

' ByRef：参照を渡す（元の変数も変わる）← 意図しない変更に注意
Sub テスト()
    Dim x As Long
    x = 5
    Debug.Print 2倍にする(x)   ' → 10
    Debug.Print x               ' ByVal なら 5 のまま / ByRef なら 10 に変わっている
End Sub
```

**▶ 話すこと**
- 「迷ったら `ByVal`」を繰り返し伝える。
- `ByRef` は「意図して元の変数を変えたいとき」だけ使う。

### スライド 15｜シートの参照方法（安全な書き方）

```vba
' NG：ActiveSheet はどのシートが選ばれているか保証がない
ActiveSheet.Cells(1, 1).Value = "タイトル"

' NG：名前が変わると動かなくなる
Worksheets("集計").Cells(1, 1).Value = "タイトル"

' OK：変数に入れてから使う ← この書き方を習慣にする
Dim ws As Worksheet
Set ws = ThisWorkbook.Worksheets("集計")
ws.Cells(1, 1).Value = "タイトル"
```

**▶ 話すこと**
- `Set` キーワードを忘れると「オブジェクト変数または With ブロック変数が設定されていません」エラーが出る。
- 「オブジェクトの代入には `Set` が必要」はルールとして覚えてもらう。

### スライド 16｜シートの作成と削除

```vba
Sub シートを準備する()
    Dim ws        As Worksheet
    Dim sheetName As String
    sheetName = "集計"

    ' ① 既存シートの確認と削除
    Application.DisplayAlerts = False     ' 削除確認ダイアログを非表示
    On Error Resume Next
    Set ws = Worksheets(sheetName)
    On Error GoTo 0

    If Not ws Is Nothing Then
        ws.Delete
        Set ws = Nothing
    End If

    ' ② 新しいシートを末尾に追加
    Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = sheetName

    Application.DisplayAlerts = True
End Sub
```

**▶ 話すこと**
- `Application.DisplayAlerts = False` を忘れると削除時に確認ダイアログが出てしまう。
- 処理が終わったら `True` に戻すことを忘れずに。

### スライド 17｜全シートをループする

```vba
Sub 全シートの名前を表示する()
    Dim ws As Worksheet
    For Each ws In ThisWorkbook.Worksheets
        Debug.Print ws.Name
    Next ws
End Sub

' 特定シートをスキップしたい場合
For Each ws In ThisWorkbook.Worksheets
    If ws.Name = "演習説明" Then GoTo NextSheet   ' スキップ
    ' 処理
NextSheet:
Next ws
```

**▶ 話すこと**
- `For Each` は「いくつあるかわからない」コレクションをループするときに使う。
- 演習4では全データシートの行をループするため、`For i = 2 To lastRow` と組み合わせる。

## 付録：講師向けトラブルシューティング

### よく出るエラーと対応

| エラーメッセージ | 原因 | 対応 |
|---|---|---|
| `オブジェクト変数または With ブロック変数が設定されていません` | `Set` を忘れている | `Set ws = Worksheets(...)` と書くよう促す |
| `インデックスが有効範囲にありません` | 存在しないシート名やインデックスを指定 | シート名のタイポを確認させる |
| `実行時エラー '1004'` | セルへのアクセスや操作が無効 | エラーが出た行をステップ実行して特定させる |
| `ファイルが見つかりません` | パスの区切り文字（`\\` vs `/`）の問題 | `Debug.Print folderPath` でパスを確認させる |
| `変数が定義されていません` | `Option Explicit` が有効なのに `Dim` を忘れた | 変数宣言を追加させる |

### よくある質問への回答

**Q：`Call` は省略できますか？**  
A：できます。`Call プロシージャ名` は `プロシージャ名` と書いても同じです。ただし `Call` をつけると「これは Sub の呼び出しだ」と明示できるので、読みやすさの観点からつけることを推奨しています。

**Q：Function と Sub はどちらを使えばいいですか？**  
A：「結果を返す必要があるか」で判断します。計算・判定など「何かを返す処理」は `Function`、実行するだけの処理は `Sub` を使います。

**Q：`On Error GoTo 0` と `On Error GoTo ErrorHandler` の違いは？**  
A：`On Error GoTo 0` は「エラーハンドリングを解除する（デフォルト状態に戻す）」コードです。`On Error Resume Next` や個別の `On Error GoTo` を使った後に、その範囲を限定するために `On Error GoTo 0` でリセットします。

**Q：`ThisWorkbook` と `ActiveWorkbook` の違いは？**  
A：`ThisWorkbook` はそのマクロが書かれているブック（固定）、`ActiveWorkbook` は現在アクティブなブック（変動する）です。外部ファイルを開く処理などでは、意図しないブックを操作してしまうリスクがあるため、基本的には `ThisWorkbook` を使うことを推奨します。
